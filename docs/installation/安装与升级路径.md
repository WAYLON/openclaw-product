# 安装与升级路径

这份文档统一覆盖 3 类场景：

- 新机器安装
- 既有机器升级
- 需要备份、清理后再重装

## 1. 新机器安装

适用于一台还没有安装过这套环境的新机器。

原则：

- OpenClaw 本体安装优先走官方文档
- 飞书插件安装优先走官方文档
- 本项目只补 Agent、插件、技能和交付结构
- 如果机器上曾装过较早期环境，先改走本文第 3 部分

推荐顺序：

1. 按官方文档安装最新稳定版 OpenClaw
2. 按官方文档安装飞书官方插件
3. 初始化本机 OpenClaw
4. 配置 `local-proxy/gpt-5.4`
5. 启用：
   - `memory-lancedb-pro`
   - `lossless-claw`
6. 创建需要的 Agent
7. 同步共享技能和专有技能
8. 绑定飞书 bot
9. 写 key
10. 执行：
    - `make delivery-check`
    - `./agent-platform acceptance --full --report ./reports/new-machine-acceptance.md`

项目命令：

```bash
make venv
./agent-platform init
./agent-platform doctor
./agent-platform agent install education-agent
./agent-platform agent install stock-agent
./agent-platform agent install loan-agent
./agent-platform agent install social-media-agent
./agent-platform agent install news-agent
./agent-platform agent install sales-agent
./agent-platform openclaw export-all --openclaw-home ~/.openclaw
make delivery-check
make acceptance
```

## 2. 既有机器升级

适用于客户已经装过较早期版本，需要升级到当前交付结构的情况。

原则：

- 升级走迁移，不走覆盖
- 不直接覆盖客户整个 `openclaw.json`
- 不直接替换客户整个 workspace
- 不顺手改其他 Agent 的模型、权限、记忆、渠道绑定

升级前先做：

1. 备份客户当前：
   - `~/.openclaw/openclaw.json`
   - 相关 workspace
   - `~/.openclaw/memory`
   - LaunchAgent key 配置
   - `~/.openclaw` 下现有 auth / sessions / pairing 相关状态
2. 检查当前机器是否已有历史 Agent 残留
3. 先清理项目内失效模板记录
4. 如果既有状态已经污染安装或运行链路，直接改走本文第 3 部分

推荐顺序：

1. 更新项目仓库
2. 执行：
   - `./agent-platform agent cleanup`
3. 安装本次新增或缺失的 Agent 技能包
4. 只同步目标 Agent，不覆盖全量客户配置
5. 补新增 key / token / access code
6. 重启 gateway
7. 执行：
   - `make delivery-check`
   - `./agent-platform acceptance --full --report ./reports/upgrade-acceptance.md`
8. 如果需要，再按 Agent 做场景验收

推荐命令：

```bash
./agent-platform agent cleanup
./agent-platform agent list
./agent-platform openclaw export-all --openclaw-home ~/.openclaw
./agent-platform acceptance
```

何时需要单独迁移：

- 新增专有技能
- 新增运行时 key
- 新增业务 Agent
- 需要替换默认模型

## 3. 备份、清理、重装

适用于这几种情况：

- 机器之前装过较早期 OpenClaw，需要彻底重装
- 当前环境疑似混入既有状态，表面安装成功但运行异常
- 需要把机器恢复到干净基线后再按当前项目安装

原则：

- 固定走：备份 -> 清理 -> 重装 -> 验收
- 不要直接在既有状态上叠加安装
- 清理前先备份客户现场配置和关键数据

### 3.1 先备份

至少备份这些内容：

- `~/.openclaw/openclaw.json`
- `~/.openclaw/workspaces`
- `~/.openclaw/memory`
- `~/Library/LaunchAgents/ai.openclaw.gateway.plist`
- `/Applications/OpenClaw.app`

建议额外记录：

- 当前 `openclaw --version`
- 当前 `openclaw status --deep`
- 当前飞书 bot 绑定关系
- 当前运行时 key 清单

### 3.2 清理既有状态

需要重点清理的是：

- 既有 `~/.openclaw` 配置目录
- 既有 Gateway LaunchAgent
- 既有 memory 库
- 既有 device pairing / auth / session 残留
- 既有 `OpenClaw.app`

推荐顺序：

1. 停止 gateway / 退出 App
2. 卸载既有 CLI / 删除既有 App
3. 卸载或移除既有 LaunchAgent
4. 备份后删除 `~/.openclaw`
5. 确认既有状态目录已经不再被复用

### 3.3 重新安装

重装动作本身优先按官方文档执行：

- OpenClaw CLI / App：先看官方文档
- 飞书官方插件：先看官方文档

然后再回到本项目文档继续：

- [7个Agent完整安装教程.md](/Users/waylon/Desktop/openclaw-product/docs/installation/7个Agent完整安装教程.md)

### 3.4 重装后的最低验收

至少要覆盖：

1. Gateway probe
2. Agent 最小回复
3. memory 健康检查
4. Feishu 配置健康检查

项目命令入口：

```bash
make delivery-check
./agent-platform acceptance --full --report ./reports/reinstall-acceptance.md
```

### 3.5 什么时候必须走这条路径

出现下面任一情况时，优先按这份文档处理：

- `pairing required`
- memory 初始化异常或索引异常
- 明明升级了但还在读既有配置
- Feishu bot 状态数量不对，例如看起来像 `6/7`
- App / CLI / Gateway 版本状态混乱
