# 8个机器人完整安装教程

> 这是一份“全量安装 8 个机器人”的方案。  
> 如果客户只需要安装其中几个，请优先阅读：
> [按需安装机器人教程.md](/Users/waylon/Desktop/openclaw-product/docs/installation/按需安装机器人教程.md)

> 目标：把 `openclaw-product` 里的 8 个专业机器人完整安装到本机 OpenClaw，并完成最小验收。  
> 当前推荐口径：**官方 OpenClaw 命令负责安装与绑定**，`agent-platform` 只负责生成项目模板和同步专业内容。

## 1. 前置条件

如果本机还没有安装 OpenClaw，请先按官方方式安装本体：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
openclaw onboard --install-daemon
```

然后确认：

- 本机已能正常执行 `openclaw`
- 项目目录已存在：
  - `/Users/waylon/Desktop/openclaw-product`
- 项目虚拟环境已准备好：
  - `/Users/waylon/Desktop/openclaw-product/.venv`

快速检查：

```bash
openclaw --version
openclaw status
cd /Users/waylon/Desktop/openclaw-product
./agent-platform doctor
```

成功标准：

- `openclaw --version` 正常返回
- `openclaw status` 正常返回
- `./agent-platform doctor` 返回 `检查通过`

## 2. 先用官方命令注册 8 个 Agent

先按官方方式把 8 个 Agent 注册到本机 OpenClaw：

```bash
for agent in \
  stock-agent \
  education-agent \
  news-agent \
  loan-agent \
  local-commerce-agent \
  photo-polisher \
  sales-agent \
  legal-agent
do
  openclaw agents add "$agent"
done
```

如果你已经注册过，可以跳过这一步。

## 10. 如果要继续绑定飞书机器人

官方绑定命令是：

```bash
openclaw agents bind --agent stock-agent --channel feishu --account stockbot
```

但飞书“扫码创建新机器人”这一步，仍然需要官方插件安装器。

如果你下一步要给某个机器人绑定独立飞书机器人，执行：

```bash
python3 /Users/waylon/.openclaw/workspace/feishu_create_and_bind_bot.py \
  --account-id stockbot \
  --name 股票机器人 \
  --agent stock-agent
```

这条命令会自动完成：

1. 启动飞书官方安装器
2. 自动选择“不复用现有机器人”
3. 等你扫码
4. 新机器人创建完成后自动沉淀为独立 `accountId`
5. 恢复原 `default`
6. 绑定到指定机器人
7. 重启 OpenClaw gateway

## 11. 常见报错排查

### 9.1 `缺少 PyYAML`

说明你用系统 Python 跑了 CLI，没有走项目虚拟环境。

先执行：

```bash
cd /Users/waylon/Desktop/openclaw-product
make venv
```

现在 `./agent-platform` 已经会优先切到项目 `.venv`，正常情况下不用手动切 Python。

### 11.2 `openclaw agents list` 看不到新机器人

先检查是否真的执行了导出：

```bash
cat /Users/waylon/.openclaw/parallel-agents/registry.json
```

再重新执行：

```bash
cd /Users/waylon/Desktop/openclaw-product
./agent-platform openclaw export <agent>
```

如果这个 Agent 还没用官方命令注册过，再补：

```bash
openclaw agents add <agent>
openclaw agents set-identity --agent <agent> --identity-file /Users/waylon/.openclaw/parallel-agents/workspaces/<agent>/IDENTITY.md
```

### 11.3 最小调用失败

先检查 OpenClaw 本体是否在线：

```bash
openclaw status
```

再检查默认模型是否可用：

```bash
openclaw agent --agent stock-agent --message "回复 OK" --json
```

### 11.4 `openclaw: command not found`

说明 OpenClaw 本体还没有装好，或者当前终端没有加载到 OpenClaw 的 PATH。

重新执行官方安装：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
openclaw onboard --install-daemon
```

再重新打开一个终端，执行：

```bash
openclaw --version
```

## 12. 推荐执行顺序

建议严格按这个顺序：

1. `curl -fsSL https://openclaw.ai/install.sh | bash`
2. `openclaw onboard --install-daemon`
3. `openclaw agents add <8个机器人>`
4. `./agent-platform init`
5. `./agent-platform start`
6. `./agent-platform agent install <8个机器人>`
7. `./agent-platform openclaw export <8个机器人>`
8. `openclaw agents set-identity <8个机器人>`
9. `openclaw agents list`
10. 最小调用验收
11. `openclaw agents bind ...`
12. 再去做飞书扫码创建等插件侧动作
