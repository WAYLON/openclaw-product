# 平台安装教程

> 目标：先按官方方式装好 OpenClaw 本体，再把本仓库作为 Agent 内容模板仓库接入本机环境。

## 1. 环境要求

### 1.0 安装前准备
- 建议先准备好 OpenClaw 运行时依赖的模型与网络环境：
  - Codex Proxy 项目：
    [icebear0828/codex-proxy](https://github.com/icebear0828/codex-proxy)
  - 网络服务购买入口：
    [WgetCloud 用户中心](https://b8cfff2a4jquxdbmwbaj.wgetcloud.org/user/shop)
- 如需在现场快速补齐基础环境，可先准备桌面目录中的本地安装包：
  - [Clash.Verge_2.4.6_aarch64.dmg](/Users/waylon/Desktop/openclaw/mac/Clash.Verge_2.4.6_aarch64.dmg)
  - [googlechrome.dmg](/Users/waylon/Desktop/openclaw/mac/googlechrome.dmg)
- 如果你计划让 OpenClaw 通过本地代理模式访问上游模型服务，建议先阅读并准备好 `codex-proxy`。
- 如果当前网络环境无法稳定访问上游服务，建议先处理网络线路，再继续执行 OpenClaw 安装。

### 1.1 操作系统
- macOS 13+
- Ubuntu 22.04+
- 其他 Linux 发行版也可，但需要能运行 `python3`

### 1.2 基础依赖
- `git`
- `python3 >= 3.11`
- `bash` 或 `zsh`

### 1.3 建议环境
- 已准备好 Codex / GPT-5.4 默认模型接入方式
- 网络可访问企业内部依赖或外部 API

## 2. 先按官方方式安装 OpenClaw 本体

如果本机还没有安装 OpenClaw，请先执行官方安装。

### 2.1 安装命令

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

### 2.2 完成官方初始化

```bash
openclaw onboard --install-daemon
```

如果你希望后续通过桌面控制台查看状态，也可以执行：

```bash
openclaw dashboard
```

### 2.3 验证 OpenClaw 本体安装成功

```bash
openclaw --version
openclaw status
```

成功标准：

- `openclaw --version` 正常返回版本号
- `openclaw status` 能正常输出状态页
- gateway 没有报 `unreachable`

## 3. 拉代码

```bash
cd ~/Desktop
git clone https://github.com/WAYLON/openclaw-product
cd openclaw-product
```

如果代码已经存在，更新方式：

```bash
cd ~/Desktop/openclaw-product
git pull
```

## 4. 安装本仓库依赖

本仓库只负责模板生成、同步和交付文档，先检查环境：

```bash
python3 --version
git --version
```

建议创建虚拟环境，但当前 CLI 不强制：

```bash
python3 -m venv .venv
source .venv/bin/activate
```

## 5. 初始化项目模板状态

在仓库根目录执行：

```bash
./agent-platform init
```

预期输出：

```text
平台初始化完成
状态目录: /你的路径/openclaw-product/.platform
```

初始化后会生成：
- `.platform/platform.json`
- `.platform/agents/`
- `.platform/workspaces/`
- `.platform/secrets/`

## 6. 启动项目状态

当前 `start` 只是给项目自己的模板状态落盘，不会替代 OpenClaw daemon：

```bash
./agent-platform start
```

查看项目状态：

```bash
./agent-platform status
```

预期输出包含：
- `started: true`
- `mode: mvp-skeleton`

## 7. 验证项目模板是否可用

### 7.1 运行项目自检

```bash
./agent-platform doctor
```

预期输出：

```text
检查通过
- agent 数量: 8
- docs 目录: True
```

### 7.2 检查关键文件

```bash
ls docs/architecture
ls agents
ls packages
```

### 7.3 检查 CLI 是否可执行

```bash
./agent-platform --help
```

## 8. 常见报错排查

### 报错：`Permission denied`

执行：

```bash
chmod +x ./agent-platform
```

### 报错：`python3: command not found`

说明机器没有 Python 3，请先安装 Python 3.11 或以上版本。

### 报错：`未找到 Agent 包`

说明你当前目录不对，或者 `packages/<agent>` 没生成。先确认：

```bash
pwd
find packages -maxdepth 1 -type d
```

### 报错：`平台尚未启动`

先执行：

```bash
./agent-platform init
./agent-platform start
```

### 报错：`openclaw: command not found`

说明 OpenClaw 本体还没有正确安装，或者当前 shell 没有加载到 PATH。

先重新执行官方安装：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
openclaw onboard --install-daemon
```

如果是新开的终端仍然找不到命令，重新打开终端后再试：

```bash
openclaw --version
```

### 报错：`Gateway unreachable`

先检查 OpenClaw 本体是否已完成初始化：

```bash
openclaw status
```

如果还没初始化，重新执行：

```bash
openclaw onboard --install-daemon
```

## 9. 安装成功判定标准

满足以下条件即可认为“OpenClaw 本体 + 本仓库模板层”安装成功：
- `openclaw --version` 成功
- `openclaw status` 成功
- `./agent-platform init` 成功
- `./agent-platform start` 成功
- `./agent-platform doctor` 成功
- `agents/` 下可见 8 个 Agent
- `packages/` 下可见 8 个交付包
