# 平台安装教程

> 目标：先按官方方式装好 OpenClaw 本体，再把本仓库作为当前 7 个首发 Agent 的交付工程接入本机环境。

当前平台口径：

- 运行态：`main + 6` 个业务 Agent
- 默认模型：`local-proxy / gpt-5.4`
- 备用大模型：`siliconflow / Qwen/Qwen3.5-122B-A10B`
- 飞书：`7` 个 bot 对 `7` 个 Agent

## 1. 安装 OpenClaw

安装口径：

- OpenClaw 本体先看官方文档
- 飞书插件先看官方文档
- 本文只保留当前项目已验证可用的官方命令入口
- `npm install -g openclaw` 只处理 CLI，不会自动下载或更新 `OpenClaw.app`
- `npx -y @larksuite/openclaw-lark install` 只处理飞书插件，不会自动处理 App
- `OpenClaw.app` 需要单独按官方文档安装或升级

```bash
npm install -g openclaw
openclaw onboard --mode local --flow quickstart --non-interactive --accept-risk --install-daemon --skip-channels --skip-search --skip-ui --skip-health --auth-choice skip
npx -y @larksuite/openclaw-lark install
```

验证：

```bash
openclaw --version
openclaw status --deep
```

## 5.1 Chrome 远程调试要求

`opencli`、`agent-browser` 这类技能依赖 Chrome 远程调试。

安装后需要额外确认：

1. 启动带远程调试端口的 Chrome
2. 打开：
   - `chrome://inspect/#remote-debugging`
3. 确认可见当前本机 Chrome 调试目标

如果没有开启这一步，很多网页浏览、登录态读取、页面自动化能力会表现成“技能已安装，但实际不可用”。

## 6.1 可选平台插件：Proactive Engine

插件页：

- [Proactive Engine](https://clawhub.ai/plugins/proactive-engine)

官方安装命令：

```bash
openclaw plugins install clawhub:proactive-engine
```

当前现场结论：

- 这是一个可选平台插件，不属于默认首发基线
- 当前这台机器安装失败，原因是：
  - 插件要求 `plugin API 2026.3.28`
  - 当前 OpenClaw runtime 暴露的是 `2026.3.25`
- 所以当前项目只把它记为“可选插件”，不写成默认安装项

用途：

- 周期性上下文检查
- 主动写入记忆
- 配合 cron 做主动巡检

## 7. 关键规则

- 运行时 key：
  - LaunchAgent：需要
  - 环境变量：必须写明变量名
- 配置型 skill key：
  - LaunchAgent：不需要
  - 环境变量：不使用
  - 配置位置：写 `openclaw.json`
- 一个飞书 bot 只绑定一个 Agent
- 群聊只在 `@机器人` 时回复
- 安装动作优先以官方文档为准，本项目文档只补当前交付结构
