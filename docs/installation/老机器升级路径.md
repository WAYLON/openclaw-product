# 老机器升级路径

> 适用于客户已经装过较早期版本，需要升级到当前 7 角色结构的情况。

## 原则

- 升级走迁移，不走覆盖
- 不直接覆盖客户整个 `openclaw.json`
- 不直接替换客户整个 workspace
- 不顺手改其他 Agent 的模型、权限、记忆、渠道绑定

## 升级前先做

1. 备份客户当前：
   - `~/.openclaw/openclaw.json`
   - 相关 workspace
   - `~/.openclaw/memory`
   - LaunchAgent key 配置
   - `~/.openclaw` 下现有 auth / sessions / pairing 相关状态
2. 检查当前机器是否已有历史角色残留
3. 先清理项目内失效模板记录
4. 如果既有状态已经污染安装或运行链路，直接改走：
   - [重装与清理路径.md](/Users/waylon/Desktop/openclaw-product/docs/installation/重装与清理路径.md)

## 推荐顺序

1. 更新项目仓库
2. 执行：
   - `./agent-platform agent cleanup`
3. 安装本次新增或缺失的 Agent 模板
4. 只同步目标 Agent，不覆盖全量客户配置
5. 补新增 key / token / access code
6. 重启 gateway
7. 执行：
   - `make delivery-check`
   - `./agent-platform acceptance --full --report ./reports/upgrade-acceptance.md`
8. 如果需要，再按角色做场景验收

## 推荐命令

```bash
./agent-platform agent cleanup
./agent-platform agent list
./agent-platform openclaw export-all --openclaw-home ~/.openclaw
./agent-platform acceptance
```

## 何时需要单独迁移

- 新增专有技能
- 新增运行时 key
- 新增业务 Agent
- 需要替换默认模型
