# 重装与清理路径

> 适用于这几种情况：
>
> - 机器之前装过旧版 OpenClaw，需要彻底重装
> - 当前环境疑似混入旧状态，表面安装成功但运行异常
> - 需要把机器恢复到干净基线后再按当前项目安装

## 原则

- 固定走：备份 -> 清理 -> 重装 -> 验收
- 不要直接在旧状态上叠加安装
- 清理前先备份客户现场配置和关键数据

## 1. 先备份

至少备份这些内容：

- `~/.openclaw/openclaw.json`
- `~/.openclaw/workspaces`
- `~/.openclaw/memory`
- `~/Library/LaunchAgents/ai.openclaw.gateway.plist`
- `/Applications/OpenClaw.app`

建议额外记录：

- 当前 `openclaw --version`
- 当前 `openclaw status --deep`
- 当前飞书 bot 绑定关系
- 当前运行时 key 清单

## 2. 清理旧状态

需要重点清理的是：

- 旧 `~/.openclaw` 配置目录
- 旧 Gateway LaunchAgent
- 旧 memory 库
- 旧 device pairing / auth / session 残留
- 旧 `OpenClaw.app`

推荐顺序：

1. 停止 gateway / 退出 App
2. 卸载旧 CLI / 删除旧 App
3. 卸载或移除旧 LaunchAgent
4. 备份后删除 `~/.openclaw`
5. 确认旧状态目录已经不再被复用

## 3. 重新安装

重装动作本身优先按官方文档执行：

- OpenClaw CLI / App：先看官方文档
- 飞书官方插件：先看官方文档

然后再回到本项目文档继续：

- [新机器安装路径.md](/Users/waylon/Desktop/openclaw-product/docs/installation/新机器安装路径.md)
- [7个Agent完整安装教程.md](/Users/waylon/Desktop/openclaw-product/docs/installation/7个Agent完整安装教程.md)

## 4. 重装后的最低验收

至少要覆盖：

1. Gateway probe
2. 7 个 Agent 最小回复
3. memory 健康检查
4. Feishu 配置健康检查

项目命令入口：

```bash
make delivery-check
./agent-platform acceptance --full --report ./reports/reinstall-acceptance.md
```

## 5. 什么时候必须走这条路径

出现下面任一情况时，优先按这份文档处理：

- `pairing required`
- memory 初始化异常或索引异常
- 明明升级了但还在读旧配置
- Feishu bot 状态数量不对，例如看起来像 `6/7`
- App / CLI / Gateway 版本状态混乱
