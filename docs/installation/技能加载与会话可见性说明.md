# 技能加载与会话可见性说明

> 这份文档用于说明：共享技能、专有技能以及会话快照之间到底是什么关系。  
> 项目安装后，判断一个 Agent “会不会用某个技能”，不能只看磁盘文件是否存在。

## 1. 两类技能的来源

### 共享技能

共享技能不放在项目仓库的 `agents/<agent>/skills/` 里。

它们来自实际运行环境的共享目录，例如：

- `~/.openclaw/skills`

这类技能的特点：

- 由平台统一安装
- 多个 Agent 可共用
- 是否可见，取决于运行时是否真的装进 OpenClaw，以及当前会话是否已经加载

### 专有技能

专有技能只属于某个 Agent，自身定义保存在项目仓库：

- `agents/<agent>/skills/`

这类技能的特点：

- 只属于目标 Agent
- 通过项目安装器同步到目标 workspace
- 最终是否可见，也仍然要看当前会话快照

## 2. 共享技能怎么进入 Agent

共享技能的加载链路是：

1. 先安装到 OpenClaw 的共享技能目录
2. gateway 启动时装载共享技能
3. Agent 新会话创建时，把共享技能纳入当前会话可见能力
4. 当前轮真正能否使用，以该会话快照为准

所以：

- 共享技能安装成功  
  不等于  
- 某个旧会话里已经能看到它

## 3. 专有技能怎么进入 Agent

专有技能的加载链路是：

1. 项目仓库里存在 `agents/<agent>/skills/<skill-name>/`
2. 安装器把它同步到目标 OpenClaw workspace
3. gateway 读取该 Agent workspace
4. 新会话创建时，把该 Agent 自己的专有技能纳入当前会话快照

所以：

- 项目里有这个 skill  
  不等于  
- 当前运行中的旧会话已经能看到它

## 4. 为什么“磁盘里有 skill”但运行时还说没接通

因为运行时真正决定可见性的，是**会话快照**，不是磁盘文件本身。

最常见的几种情况：

1. skill 文件已经放对了
2. gateway 也已经重启了
3. 但当前 Agent 还在沿用既有 session
4. 既有 session 里的技能快照没有更新

结果就是：

- 文件已经在
- 运行时还会说“没接通”或“看不到这个技能”

## 5. 正确验证方法

不要只验证：

- 文件在不在
- `openclaw skills list` 有没有

真正要验证的是：

1. 目标 Agent 开新会话
2. 用标准问题触发该技能
3. 看它是否按该技能处理请求

## 6. 现场排障顺序

当某个技能“看起来装了但 Agent 说没看到”时，推荐按下面顺序排：

1. 先确认 skill 来源对不对
   - 共享技能：看共享目录
   - 专有技能：看该 Agent 的 `skills/`
2. 再确认安装器是否已经同步到目标 workspace
3. 再确认 gateway 是否已重启
4. 最后优先处理 session snapshot
   - 开新会话
   - 必要时重建目标 Agent 的 session snapshot

## 7. 当前项目的正式口径

- 项目仓库里的 `agents/<agent>/skills/`
  - 只表达专有技能定义
- 共享技能
  - 由实际 OpenClaw 环境统一安装
- 最终运行态是否真的能用
  - 以当前 Agent 会话可见性为准

一句话：

- 共享技能看运行时共享目录
- 专有技能看目标 Agent 的 `skills/`
- 最后都要以**会话快照**决定是否真正可用
