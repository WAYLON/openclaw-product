# 记忆插件安装教程

> 当前项目正式口径里的两款插件只有：
>
> - `memory-lancedb-pro`
> - `lossless-claw`
>
> 旧的 `benos-memory-core` 只属于历史模板残留，不再作为当前首发安装基线。

## 1. 角色定位

- `memory-lancedb-pro`
  - 负责长期记忆
  - 保存和召回跨会话的偏好、事实、项目背景
- `lossless-claw`
  - 负责上下文压缩
  - 在当前会话过长时保留结构化摘要

这两个插件分工不同，可以同时启用。

## 2. 当前首发安装口径

当前项目以这台机器的真实运行态为准：

- `plugins.slots.memory = memory-lancedb-pro`
- `plugins.slots.contextEngine = lossless-claw`

## 3. memory-lancedb-pro 安装

### 3.1 安装命令

```bash
curl -fsSL https://raw.githubusercontent.com/CortexReach/toolbox/main/memory-lancedb-pro-setup/setup-memory.sh -o setup-memory.sh
bash setup-memory.sh
```

### 3.2 启用要求

- 需要 embedding provider
- 当前已验证通过的是：
  - `provider = openai-compatible`
  - `baseURL = https://api.siliconflow.cn/v1`
  - `model = Qwen/Qwen3-Embedding-8B`

### 3.3 运行时 key 规则

如果 embedding 或插件运行依赖后台 runtime key，则统一写入：

- `~/Library/LaunchAgents/ai.openclaw.gateway.plist`

然后：

1. reload LaunchAgent
2. restart gateway
3. 验证运行时环境已生效

## 4. lossless-claw 安装

### 4.1 安装来源

- 当前实际来源：
  - [lossless-claw-enhanced](https://github.com/win4r/lossless-claw-enhanced)

### 4.2 启用口径

- `plugins.slots.contextEngine = lossless-claw`

### 4.3 使用原则

- 它只负责上下文压缩
- 不替代长期记忆插件
- 不单独声明成业务专有能力

## 5. 验证项

至少验证：

1. `openclaw status --deep`
2. `openclaw memory-pro stats`
3. 最小回复链路可正常返回

## 6. 当前项目约束

- 这两款插件属于平台级插件，不属于某个业务 Agent 的专有技能
- 它们写进项目安装基线，但不写成某个业务 Agent 的“专有 skill”
- 后续如果替换插件，必须同步更新：
  - `README.md`
  - 架构文档
  - 安装教程
  - 发布流程
