# 渠道绑定教程

> 原则：一个渠道实例只绑定一个 Agent，不做业务总控分发。  
> 当前推荐口径：**官方 OpenClaw 命令和官方插件优先**，真实绑定只使用 `openclaw agents bind`。

## 1. 官方优先绑定命令

如果是本机真实 OpenClaw 环境，优先使用：

```bash
openclaw agents bind --agent <agent> --channel <channel> --account <account-id>
```

查看绑定：

```bash
openclaw agents bindings --json
```

## 2. 飞书绑定

### 2.0 两种绑定路径

飞书绑定现在分两种：

1. 平台抽象写法
   - 适合做文档、方案和安装流程演示
2. 今天本地实际跑通写法
   - 适合真实交付
   - 基于 `npx -y @larksuite/openclaw-lark install`
   - 基于本机 `openclaw agents bind`
   - 基于 [feishu_capture_new_bot.py](/Users/waylon/.openclaw/workspace/feishu_capture_new_bot.py)
   - 基于 [feishu_create_and_bind_bot.py](/Users/waylon/.openclaw/workspace/feishu_create_and_bind_bot.py)

### 2.1 绑定命令

```bash
openclaw agents bind --agent education-agent --channel feishu --account educationbot
```

### 2.2 配置文件示例

参考：
[channel-binding.example.yaml](/Users/waylon/Desktop/openclaw-product/docs/templates/channel-binding.example.yaml)

飞书实例示例：

```yaml
channel_instance:
  id: feishu-edu-bot
  type: feishu
  tenant_id: demo-tenant
  bound_agent_id: education-agent
  reply_language: zh-CN
  init_prompt_template: default_cn
```

### 2.3 验证消息打通

检查绑定：

```bash
openclaw agents bindings --json
```

确认输出里有：

```text
education-agent <- feishu:educationbot
```

### 2.4 今天真实可执行的本地飞书绑定流程

#### 第一步：启动飞书官方安装器

```bash
npx -y @larksuite/openclaw-lark install --verbose
```

#### 第二步：不要复用旧机器人

如果看到：

```text
Found Feishu bot with App ID ...
Use it for this setup?
```

必须选：

```text
n
```

#### 第三步：新建机器人并扫码

安装器会打印一条新建链接，类似：

```text
https://open.feishu.cn/page/openclaw?user_code=XXXX-XXXX&from=onboard
```

建议做法：

- 把这个链接打开成浏览器大页
- 不要直接扫终端里的小二维码

#### 第四步：把新机器人沉淀成独立 accountId

飞书官方安装器默认会把新机器人写进 `default`。  
正确做法是立刻执行：

```bash
python3 /Users/waylon/.openclaw/workspace/feishu_capture_new_bot.py \
  --account-id educationbot \
  --name 教育机器人 \
  --agent education-agent
```

这个动作会：

- 把新机器人保存为 `educationbot`
- 恢复旧 `default`
- 自动绑定到 `education-agent`
- 重启 gateway

#### 第五步：验证

```bash
openclaw channels status --probe --json
openclaw agents bindings --json
```

如果成功，应能看到：

```text
education-agent <- feishu:educationbot
```

## 7. 常见报错排查

### 报错：`新机器人把 default 覆盖了`

这是飞书官方安装器的默认行为，不是异常。  
正确处理方式不是手改配置，而是运行：

```bash
python3 /Users/waylon/.openclaw/workspace/feishu_capture_new_bot.py \
  --account-id <new-account-id> \
  --name <中文名> \
  --agent <agent-id>
```

### 报错：`Agent 未安装`

先执行官方命令：

```bash
openclaw agents add <agent>
```

### 报错：`不支持的渠道类型`

当前支持：
- `feishu`
- `wecom`
- `dingtalk`
- `webhook`
- `webchat`
- `api`
- `wechat`

### 重复绑定同一个实例

同一个 `instance-id` 会被新内容覆盖，建议统一命名规范，例如：
- `feishu-edu-bot`
- `wecom-stock-app`
- `photo-webhook`
