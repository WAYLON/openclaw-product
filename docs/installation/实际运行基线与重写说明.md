# 实际运行基线与重写说明

这份文档用于说明：

- 当前 `openclaw-product` 仓库已经按哪套真实运行环境重写
- 哪些配置以桌面上的实际安装文档为准
- 新机器要如何复刻出相同结构

## 1. 真实基线来源

当前项目默认以这两份文档为最终安装基线：

- `/Users/waylon/Desktop/OpenClaw-安装部署指南-精简版.md`
- `/Users/waylon/Desktop/OpenClaw-安装与排障记录.md`

本仓库中的所有安装、模板、架构口径，应优先服从这两份文档。

## 2. 当前真实运行环境

- OpenClaw CLI / App：安装口径默认取最新稳定版
- Gateway：`~/Library/LaunchAgents/ai.openclaw.gateway.plist`
- 默认模型：`local-proxy/gpt-5.4`
- 备用大模型：`siliconflow/Qwen/Qwen3.5-122B-A10B`
- 模型接口：`http://127.0.0.1:8080/v1`
- 运行模式：`local`
- 当前项目首发口径为 `7` 个 Agent / `7` 个 bot
- 记忆插件：`memory-lancedb-pro`
- 上下文压缩插件：`lossless-claw`

补充：

- 这套工程最初重写和验证时，最初验证版本曾是 `2026.3.28`
- 但当前给新机器安装时，不再要求固定这个版本号
- 当前同时保留一个硅基流动备用大模型：
  - `siliconflow / Qwen/Qwen3.5-122B-A10B`
  - `baseUrl = https://api.siliconflow.cn/v1`

说明：

- `main` 额外挂 `skill-creator`
- 其余 Agent 走统一共享能力基线
- 当前全部 Agent 为 `tools.profile = full`
- 当前项目文档口径的 `exec` 默认放权写法为：
  - `tools.profile = "full"`
  - `tools.exec.security = "full"`
  - `tools.exec.ask = "off"`
- 默认同时要求：
  - `agents.defaults.compaction.reserveTokensFloor = 20000`
- 用于避免默认会话过早触发 `Context limit exceeded`
- `opencli`、`agent-browser` 额外依赖 Chrome 远程调试
- 安装后需要在 `chrome://inspect/#remote-debugging` 确认调试目标可见

## 5. Key 规则

### 5.1 写 LaunchAgent 的

凡是给 OpenClaw 后台 gateway 运行时使用的 key，都要写到：

- `~/Library/LaunchAgents/ai.openclaw.gateway.plist`

然后重载 / 重启 gateway。

已按这个规则处理过的包括：

- `QVERIS_API_KEY`
- `TRADINGAGENTS_TOKEN`
- `GEMINI_API_KEY`

### 5.2 写 `openclaw.json` 的

如果 skill 明确要求走配置读取，则写到：

- `~/.openclaw/openclaw.json`

当前例子：

- `openmaic`
  - `skills.entries.openmaic.config.accessCode`
