# 产品包规范

每个产品包都必须遵循统一规范，这样才能持续维护和批量交付。

## 最小结构

```text
<pack>/
  README.md
  workspace/
  skills/
```

## README 必须回答的问题

1. 这个包是干什么的
2. 适合什么客户
3. 支持哪些安装方式
4. 需要客户本地补什么配置
5. 安装后如何验证

## workspace 的职责

- 定义人格模式
- 定义工作区规则
- 定义本机占位模板

典型文件：

- `SOUL.md`
- `AGENTS.md`
- `TOOLS.md`
- `USER.md`
- `MEMORY.md`

## skills 的职责

- 定义具体能力
- 定义任务路由
- 定义执行或参考资料

## 安装模式

统一使用两个模式：

- `skills-only`
- `full`

不要为不同包发明不同名字。

## 安全要求

- 不提交真实密钥
- 不提交客户私有资料
- 所有示例配置都应使用占位符
