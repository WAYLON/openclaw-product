# 初始化与人格塑造

这个文档用于说明如何把一个新安装的 OpenClaw 初始化成你想要的小龙虾。

## 一、为什么初始化重要

初始化决定了小龙虾的：

- 名字
- 身份
- 服务对象
- 性格
- 工作风格
- 长期记忆起点

如果这一步做得随意，后续工作区规则再强，也容易跑偏。

## 二、初始化时建议告诉小龙虾什么

第一句话建议包含：

```text
你好，初始化
```

接着告诉它下面这些信息：

- 你的名字
- 我的名字
- 你的性格特点
- 你能干什么
- 你的记忆

## 三、标准初始化模板

你可以直接按下面结构来写：

```text
你好，初始化

- 你的名字：XXX
- 我的名字：XXX
- 你的性格特点：XXX
- 你能干什么：XXX
- 你的记忆：XXX
```

## 四、初始化完成后应该发生什么

初始化成功后，通常应更新这些文件：

- `IDENTITY.md`
- `USER.md`
- `SOUL.md`
- `AGENTS.md`

如果存在 `BOOTSTRAP.md`，一般会被删除，表示“出生流程”已经完成。

## 五、建议的检查方式

初始化后，建议让它总结一次：

```text
请总结你现在的名字、身份、性格、职责、服务对象和工作方式。
```

如果它总结不清楚，说明初始化还不够完整。

## 六、行业包里的做法

对于本仓库中的客户专版，建议每个包自带：

- `workspace/INIT_PROMPT.md`

例如：

- `stocks/openclaw-stock-pack-langlang/workspace/INIT_PROMPT.md`

这样客户不需要自己现场编提示词。

## 七、初始化的产品化意义

初始化不是“聊两句”，而是产品交付的一部分。

一份好的初始化模板，本质上就是：

- 人格配置
- 工作模式配置
- 客户偏好注入

这也是 OpenClaw 客户专版的重要价值来源。
