# 接入国外模型说明

这个文档给内部技术安装人员使用。

目标是说明：

- 什么时候需要接国外模型
- 目标机器需要准备什么前置条件
- 安装人员需要提醒客户什么风险

## 一、什么时候会用到国外模型

以下场景通常会考虑接国外模型：

- 希望使用 OpenAI、Anthropic 等国外模型
- 需要更强的英文理解或更稳定的特定能力
- 客户明确指定某个国外模型

## 二、前置条件

如果目标机器要接国外模型，安装人员应先确认：

1. 目标机器具备可用的国际网络访问条件
2. 目标机器具备对应模型账户、API Key 或可用的中转服务
3. 客户接受相关成本和稳定性风险

一句话：

**接国外模型，通常既需要网络条件，也需要模型采购或账号准备。**

## 三、网络条件说明

在国内网络环境下，很多国外模型服务无法直接稳定访问。

因此交付时通常需要：

- 可用的国际网络接入
- 可用的代理或网络加速方案

## 四、模型采购说明

国外模型常见获取方式包括：

1. 客户自行提供官方 API Key
2. 客户自行提供中转平台 Key
3. 安装人员协助客户选择第三方采购渠道

优先建议：

- 官方渠道
- 可信中转平台

如果客户明确接受第三方采购方式，安装人员可以把下列链接作为“客户自行甄别风险的备选渠道”：

- [闲鱼示例 1](https://m.tb.cn/h.iTPk7yf?tk=rniRUzDXbbR)
- [闲鱼示例 2](https://m.tb.cn/h.igy2TQX?tk=F4QEUzDeY0f)

注意：

- 上述链接属于第三方市场链接
- 我们不把它们视为官方渠道
- 安装人员不能把这类渠道表述成“平台官方合作”
- 账号、额度、稳定性、售后都需要客户自行承担选择风险

## 五、交付时怎么说

安装人员可以统一这样说：

- 如果要接国外大模型，目标机器通常需要具备可用的国际网络访问条件
- 模型账号或 API Key 需要客户自行准备，或由客户确认接受第三方采购渠道
- 我们负责把 OpenClaw 接上对应模型，不负责保证第三方采购渠道的长期稳定性

## 六、仓库中的定位

这部分内容属于：

- 内部交付说明

不属于：

- 默认基础技能
- 默认行业包能力
- 仓库内置账号资源

## 七、安装人员检查项

安装国外模型前，至少确认：

- 网络访问条件是否具备
- API Key 是否已拿到
- OpenClaw 模型配置是否已写入
- 客户是否接受对应费用
- 是否需要备用模型

## 八、一句话总结

接国外模型不是“仓库里多装一个 skill”就完了，而是：

**网络条件 + 账号 / Key + OpenClaw 配置 三者都要具备。**
