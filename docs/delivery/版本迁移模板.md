## 4. 迁移步骤

1. 备份当前 `openclaw.json`
2. 记录当前目标 Agent 的模型、技能、渠道绑定、memory namespace
3. 拷贝新 skill 到目标 Agent 目录
4. 写入所需 key / 配置
5. 先不启用，执行依赖检查
6. 在目标 Agent 的 skill 条目中追加新 skill
7. 重启 gateway
8. 执行最小功能验证
9. 执行真实场景验证

## 5. 验证步骤

- 最小测试指令：
- 预期结果：
- 日志位置：
- 失败时判断点：
- 是否影响旧链路：

## 6. 回滚步骤

1. 从目标 Agent 的 skill 条目移除新 skill
2. 保留目录但停用
3. 重启 gateway
4. 重新验证旧链路

## 7. 注意事项

- 不改已有 `agent id`
- 不改已有 `memory namespace`
- 不改已有渠道绑定
- 不强制覆盖用户自定义配置
- 不顺手改全局共享技能分配
- 不顺手改其他 Agent 的模型或权限

## 8. 每个 Agent 的迁移提示

- `stock-agent`
  - 保留现有 `tradingagents-analysis`
  - 新 skill 默认作为增量能力
- `education-agent`
  - 保留现有 `openmaic`
  - 新 skill 不覆盖现有 `accessCode` 配置
- `loan-agent`
  - 先验证敏感材料处理边界
- `social-media-agent`
  - 先验证平台登录态、权限和网页链路
- `news-agent`
  - 先验证来源可靠性和摘要结果
- `sales-agent`
  - 先验证 CRM 或外部写入是否真实可用
