# 角色验收模板

> 这份文档用于做“按角色的最小场景验收”。它不是安装步骤，而是交付前确认每个角色至少能完成什么。

## 使用原则

- 先做平台级验收
  - `make delivery-check`
  - `make acceptance`
- 再做角色级验收
- 没有真实接通的能力，不列入“必须通过”的验收项

## 平台级前置验收

- `openclaw status --deep`
- `openclaw models status`
- 如有需要，再执行：
  - `openclaw channels status --probe`

## 1. 主入口 Agent

- 目标：
  - 验证默认入口和技能管理员职责
- 最小验收问题：
  - “给全部 agent 安装一个共享 skill 时，应该怎么做？”
  - “这个 skill 安不安全？”
- 预期：
  - 能说明共享 skill / 专有 skill 区别
  - 能说明安装、分发、审查和影响范围

## 2. 教育 Agent

- 当前真实专有能力：
  - `openmaic`
- 最小验收问题：
  - “帮我把这份讲义整理成课堂重点”
- 预期：
  - 输出中文结构化讲解结果
  - 不把未接通的平台动作写成已可用

## 3. 股票 Agent

- 当前真实专有能力：
  - `tradingagents-analysis`
- 最小验收问题：
  - “分析这只股票的风险和观察点”
- 预期：
  - 输出研究框架、依据、风险和后续观察建议
  - A 股筛股场景优先走 QVeris / THS iFinD 口径

## 4. 助贷 Agent

- 当前真实专有能力：
  - 暂无
- 最小验收问题：
  - “帮我整理这段材料说明”
- 预期：
  - 输出材料说明、下一步动作和合规提醒
  - 不越界成授信审批结论

## 5. 社媒 Agent

- 当前真实专有能力：
  - 暂无
- 最小验收问题：
  - “帮我整理今天的社媒热点”
- 预期：
  - 输出热点摘要、可复用表达和分发建议
  - 不把自动发布或账号托管写成默认能力

## 6. 新闻 Agent

- 当前真实专有能力：
  - 暂无
- 最小验收问题：
  - “把今天几条新闻整理成摘要”
- 预期：
  - 输出摘要、重点变化和简报底稿
  - 不把二手消息包装成已验证事实

## 7. 销售 Agent

- 当前真实专有能力：
  - 暂无
- 最小验收问题：
  - “总结这段客户对话，给我下一步建议”
- 预期：
  - 输出当前判断、推荐回复、下一步动作
  - 不夸大承诺、不越界到自动签约

## 验收记录建议

每个角色至少记录：

- 验收时间
- 验收人
- 使用的问题
- 实际输出是否符合预期
- 是否涉及额外 key / 登录态
- 是否需要补文档或补技能说明
