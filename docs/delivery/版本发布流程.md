# 版本发布流程

## 1. 发布前检查

执行：

```bash
make quality
python3 scripts/check_all_packages.py
python3 installer/bootstrap_project.py check
./agent-platform doctor
```

## 2. 文档检查

- 检查 6 个业务 Agent 的交付文档是否完整
- 检查安装文档是否与当前命令一致
- 检查渠道绑定总表是否与当前策略一致
- 检查模型分配总表是否与当前 Agent 工作区配置说明一致
- 检查新增专有技能是否配套：
  - 专有技能新增规范
  - 版本迁移说明
  - 回滚说明

## 3. 交付包检查

- 每个 `packages/<agent>/config/install-manifest.yaml` 必须存在
- 每个包必须包含：
  - `quickstart.md`
  - `user-guide.md`
  - `trainer-guide.md`

## 4. 本机 OpenClaw 联调检查

- 至少导出 2 个 Agent 到本机 OpenClaw
- 至少验证 1 个飞书实例绑定
- 至少跑通 1 次最小请求：
  - `openclaw agent --agent <agent> --message "回复 OK" --json`

## 5. 发布建议

- 小版本：文档、模板、校验增强
- 中版本：安装流程、渠道绑定、模型策略调整
- 大版本：目录结构、Agent 结构、权限模型变化

## 6. 新增专有技能发布要求

- 新增专有技能前，先补：
  - [专有技能新增规范.md](/Users/waylon/Desktop/openclaw-product/docs/delivery/专有技能新增规范.md)
  - [版本迁移模板.md](/Users/waylon/Desktop/openclaw-product/docs/delivery/版本迁移模板.md)
  - [回滚模板.md](/Users/waylon/Desktop/openclaw-product/docs/delivery/回滚模板.md)
- 对老客户默认按“增量迁移”处理，不做全量覆盖
- 未真实接通的技能，不得写入首发能力总览
