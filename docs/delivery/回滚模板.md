# 回滚模板

> 这份模板用于新增专有技能上线后出现问题时，快速回滚到稳定状态。

## 1. 回滚对象

- 目标 Agent：
- 目标 skill：
- 当前版本：
- 回滚原因：

## 2. 回滚原则

- 只回滚本次新增 skill
- 不回滚整套平台
- 不改动其他 Agent 已稳定运行的配置

## 3. 标准回滚步骤

1. 先确认回滚只影响目标 Agent
2. 从目标 Agent 的 skill 配置中移除该 skill
3. 保留 skill 文件目录，避免直接删文件导致后续难以排查
4. 恢复旧 key / 配置引用
5. 重启 gateway
6. 执行最小验证
7. 执行旧链路验证

## 4. 最小验证

- Agent 仍能正常启动
- 原有主链路仍能正常响应
- 相关日志不再出现新 skill 报错

## 5. 记录要求

回滚后至少记录：

- 回滚时间
- 回滚操作者
- 回滚原因
- 失败现象
- 后续是否重试上线

## 6. 不该动的东西

回滚某个专有 skill 时，不应该一起动：

- 其他 Agent 的 skill 配置
- 全局共享技能目录
- 全局记忆插件槽位
- 全局上下文压缩插件槽位
- 已稳定运行的飞书绑定
