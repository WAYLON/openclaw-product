# 产品分级说明

为了避免产品定位混乱，这个仓库中的 OpenClaw 产品统一按等级划分。

## 一、为什么要分级

如果不分级，所有产品都会混成一类：

- 价格不好定
- 卖点不好讲
- 客户不知道差别
- 你自己也不好维护

所以必须把产品按能力和交付深度分开。

## 二、四级模型

### 1. 基础版

定位：

- 最轻量
- 最易复制
- 最适合引流

典型特征：

- 基础 `workspace`
- 少量核心技能
- 文档齐全
- 不强绑定特定客户

适合：

- 想先体验 OpenClaw 的客户
- 对成本敏感的客户
- 作为演示和引流产品

### 2. 增强版

定位：

- 行业能力更强
- 技能更完整
- 更接近可落地生产力

典型特征：

- 比基础版更多的技能
- 更多行业规则
- 可能带工具接入
- 仍然不绑定某个客户

适合：

- 对行业能力有更高要求的客户
- 愿意为效率和效果付费的客户

### 3. 客户专版

定位：

- 面向单个客户
- 高度贴合客户工作流

典型特征：

- 客户专属 `SOUL`
- 客户专属 `AGENTS`
- 客户专属工具接入
- 客户专属脚本和验证方式

适合：

- 已经有明确业务需求的客户
- 需要长期合作的客户

### 4. 团队版

定位：

- 多角色协作
- 多 Agent 配置
- 更复杂的工作流

典型特征：

- 不止一个 agent
- 有多角色分工
- 有协调和汇总逻辑
- 有更强的流程管理要求

适合：

- 任务复杂
- 希望形成“AI 团队”的客户

## 四、以后新增产品时怎么判断等级

### 属于基础版，如果：

- 面向广泛用户
- 不依赖客户专属资源
- 安装后就能跑基础流程

### 属于增强版，如果：

- 比基础版多出明显行业能力
- 但仍然不是客户专属

### 属于客户专版，如果：

- 明确绑定某个客户名称
- 明确接入某个客户的工具或规则

### 属于团队版，如果：

- 产品核心卖点是多 Agent 协作
- 不是单一工作区 + 单一技能集合

## 五、定价启发

虽然仓库里不一定直接写价格，但分级本身就是定价基础：

- 基础版：低价或免费
- 增强版：标准收费
- 客户专版：高价定制
- 团队版：更高价解决方案

## 六、一句话原则

分级的本质不是做分类，而是让产品：

- 更容易卖
- 更容易讲
- 更容易维护
