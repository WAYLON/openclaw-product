# 默认交付预设

为了兼顾“演示效果”和“生产安全”，本项目不建议只保留一套默认配置。

推荐统一采用两套预设：

1. `demo/open`
2. `prod/safe`

## 一、为什么要分预设

如果只保留“全开”一种默认值：

- 演示时很爽
- 生产时风险很高

如果只保留“很安全”一种默认值：

- 生产安全
- 但演示效果差、客户感知弱

所以更合理的做法是：

- 对外演示、试用、快速交付：`demo/open`
- 真正上线、长期运行：`prod/safe`

## 二、`demo/open` 预设

适用场景：

- 现场演示
- 客户试用
- 内部测试
- 快速验证体验

建议默认开启：

- `/verbose`
- `/reasoning`
- 飞书流式输出
- 更宽松的工具权限
- 更完整的工具组合

建议行为：

- 浏览器相关能力可用
- 文件读写能力可用
- 命令执行能力可用
- 消息发送能力可用

注意：

这个模式更适合“效果优先”，不适合直接当成长期生产默认值。

## 三、`prod/safe` 预设

适用场景：

- 客户正式上线
- 长期运行
- 涉及敏感数据
- 涉及高危工具

建议默认开启：

- `/verbose`
- `/reasoning`
- 飞书流式输出

但建议保留：

- 审批
- 权限边界
- 工具最小化原则
- 敏感操作确认

## 四、飞书相关默认值

无论哪种预设，建议默认开启：

```bash
openclaw config set channels.feishu.streaming true
```

原因：

- 回复体验明显更好
- 客户感知更强
- 更像真实在线助手

## 五、我们当前建议

### 对外演示和初装

默认使用：

- `demo/open`

### 正式交付和稳定运行

默认切换为：

- `prod/safe`

## 六、一句话原则

默认值不是只有一套。

**演示讲体验，生产讲边界。**
