# 技术交付说明

这个仓库的默认读者不是客户，而是：

- 技术人员
- 实施人员
- 售前技术
- 交付人员

## 语言约定

整个交付仓库默认使用中文。

除非是下列内容，否则不要主动写英文：

- shell 命令
- 文件路径
- 环境变量
- 第三方项目名
- 技术字段名

## 一、这个仓库是干什么的

它本质上是一个 OpenClaw 交付仓库，用于：

1. 安装 OpenClaw 本体
2. 安装基础技能层
3. 安装行业包
4. 安装客户定制包
5. 进行本地验证和验收

## 本体来源约定

安装人员必须先明确：

- OpenClaw 本体先走官方安装
- 我们的仓库不替代官方本体安装
- 我们的仓库负责官方本体安装完成后的交付层

## 二、这个仓库不是什么

它不是：

- 给客户直接看的产品官网
- 给客户直接读的使用手册全集
- 给客户自己随意挑目录安装的展示站

客户真正会接触的，通常是：

- 已安装好的 OpenClaw 环境
- 少量交付说明
- 必要的本地配置步骤

## 三、正确使用方式

技术人员使用时，默认顺序应是：

1. 先看 `core/`
2. 再看目标行业包
3. 再看是否需要客户定制包
4. 最后执行本地验证脚本

## 四、为什么要这么设计

因为交付本质上分三层：

1. 公共底座
2. 行业能力
3. 客户定制

如果不这样拆，后面会出现：

- 基础技能和行业技能混在一起
- 客户 A 的定制污染客户 B
- 技术人员自己都难以维护

## 五、文档应该怎么理解

这个仓库里的文档主要有两类：

### 1. 内部交付文档

给技术人员看的，例如：

- 安装步骤
- 技能清单
- 验证方式
- 安全规范
- 差异化说明

### 2. 可抽取的客户交付材料

有些文档的内容可以整理后发给客户，但不意味着整个仓库要直接给客户看。

## 六、一句话原则

这个仓库首先服务的是“会安装、会维护、会验收的人”，不是最终客户。
