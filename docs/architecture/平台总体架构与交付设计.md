# 平行 Agent 平台总体架构与交付设计

## 1. 当前真实运行架构

本项目当前不再以早期概念模板为准，而是以已经在本机实际跑通的 OpenClaw 部署为准。

真实基线：

- OpenClaw：安装口径默认取最新稳定版
- 运行模式：`local`
- Gateway：LaunchAgent 常驻
- LaunchAgent 文件：`~/Library/LaunchAgents/ai.openclaw.gateway.plist`
- 默认模型：`local-proxy/gpt-5.4`
- 备用大模型：`siliconflow/Qwen/Qwen3.5-122B-A10B`
- 当前首发 Agent：`7`
- 飞书：当前产品口径按 `7` bot 对 `7` agent 交付
- 记忆插件：`memory-lancedb-pro`
- 上下文压缩：`lossless-claw`

补充说明：

- 这套项目最初验证时使用过 `2026.3.28`
- 但当前对外安装口径不再锁死具体版本号，默认按最新稳定版安装

## 2. 架构原则

- 不设业务总控 Agent
- 业务 Agent 平行独立
- `main` 只作为默认入口与技能管理员
- 一个飞书机器人只绑定一个 Agent
- 共享 skill 与专有 skill 分层管理
- 运行时 key 统一通过 LaunchAgent 注入
- 以“实际可运行配置”为先，不再保留与现网不一致的概念口径

## 4. 模型层

### 4.1 当前默认模型

```yaml
provider: local-proxy
model: gpt-5.4
baseUrl: http://127.0.0.1:8080/v1
apiKey: pwd
api: openai-completions
```

### 4.2 为什么统一到 `local-proxy`

当前实际验证结果说明：

- `openai-codex` OAuth 链路容易遇到配额、地区、超时和 WebSocket 异常
- `local-proxy/gpt-5.4` 明显更快、更稳
- 因此项目模板默认以 `local-proxy/gpt-5.4` 为平台主模型

### 4.2.1 当前备用大模型

当前项目同时补充了一个可选大模型：

```yaml
provider: siliconflow
model: Qwen/Qwen3.5-122B-A10B
baseUrl: https://api.siliconflow.cn/v1
api: openai-completions
```

说明：

- 这条不是当前平台主默认
- 主要作为更大参数规模的可选模型备用
- 需要 `SILICONFLOW_API_KEY`

### 4.3 Agent 模型策略

当前机器上的实际运行态是：

- 平台默认模型统一为 `local-proxy/gpt-5.4`
- 平台当前额外保留一个可选大模型：
  - `siliconflow/Qwen/Qwen3.5-122B-A10B`
- 各 Agent 暂未做单独模型覆盖

因此本项目模板默认采用：

- 平台统一默认模型
- 允许后续按 Agent 增量覆盖

## 5. 技能分层

共享基础技能安装到：

- `~/.openclaw/skills`

当前共享基础技能为：

- `web-access`
- `opencli`
- `self-improving-agent`
- `skill-vetter`
- `skill-creator`
- `summarize`
- `agent-browser`
- `ontology`
- `weather`
- `document-pro`
- `github`
- `proactive-agent`
- `mcporter`
- `multi-search-engine`
- `nano-pdf`
- `nano-banana-pro`
- `excel-xlsx`
- `word-docx`
- `desktop-control`

说明：

- 这些 skill 属于共享安装
- 再由 `openclaw.json` 显式分配给各 Agent
- 当前所有 Agent 的 `tools.profile` 都是 `full`
- 当前项目文档口径的 `exec` 默认放权写法为：
  - `tools.profile = "full"`
  - `tools.exec.security = "full"`
  - `tools.exec.ask = "off"`

### 5.2 专有技能

专有 skill 安装到：

- `~/.openclaw/workspaces/<agent-id>/skills/<skill-name>`

当前已接入：

- `stock-agent`
  - `tradingagents-analysis`
- `education-agent`
  - `openmaic`
- `social-media-agent`
  - 当前首发不挂专有 skill
  - 以共享技能组装为主

### 5.3 当前推荐分层

- 共享 skill
  - 做平台通用能力
  - 搜索、浏览、文档、总结、审查、自动学习
- 专有 skill
  - 做业务增强
  - 股票分析、教育平台接入、后续行业专属能力

## 7. 飞书渠道层

### 7.1 当前绑定原则

- 一个 bot 对应一个 Agent
- 私聊允许直接回复
- 群聊只在 `@机器人` 时回复

### 7.2 当前推荐策略

```yaml
dmPolicy: open
groupPolicy: open
requireMention: true
allowFrom:
  - "*"
```

### 7.3 当前首发 bot -> agent 绑定

- `default -> main`
- `educationbot -> education-agent`
- `stockbot -> stock-agent`
- `loanbot -> loan-agent`
- `socialbot -> social-media-agent`
- `newsbot -> news-agent`
- `salesbot -> sales-agent`

## 8. Key 管理规则

这是当前项目最重要的真实规则之一。

### 8.1 后台 runtime key

只要某个 key 是给 OpenClaw 后台 gateway 用的，就必须写入：

- `~/Library/LaunchAgents/ai.openclaw.gateway.plist`

然后：

1. reload LaunchAgent
2. restart gateway
3. 再验证运行时环境

当前已经按这个规则处理过的 key 包括：

- `QVERIS_API_KEY`
- `TRADINGAGENTS_TOKEN`
- `GEMINI_API_KEY`

### 8.2 skill 配置型 key

如果某个 skill 明确通过配置文件读取，而不是环境变量读取，则写到：

- `~/.openclaw/openclaw.json`

当前已验证的例子：

- `openmaic`
  - 走 `skills.entries.openmaic.config.accessCode`

## 9. 项目目录职责

```text
openclaw-product/
├── README.md
├── agents/
├── packages/
├── channels/
├── core/
├── installer/
├── docs/
└── scripts/
```

- `agents/`
  - 业务 Agent 模板
  - 以专有 skill 和业务工作方式为中心
- `packages/`
  - 面向交付的安装清单与操作文档
- `channels/`
  - 渠道模板和接入壳层
- `core/`
  - 配置、注册、运行时、密钥处理底座
- `installer/`
  - 模板同步、导出、结构校验
- `docs/`
  - 架构、安装、交付、培训资料

## 10. 当前基线结论

当前项目默认采用下面的统一口径：

- 模型：`local-proxy/gpt-5.4`
- 运行：LaunchAgent 守护 gateway
- 共享技能：放 `~/.openclaw/skills`
- 专有技能：放 `~/.openclaw/workspaces/<agent>/skills`
- 记忆：`memory-lancedb-pro`
- 上下文压缩：`lossless-claw`
- 飞书：7 bot 对 7 agent
- key：后台走 LaunchAgent，配置型 skill 走 `openclaw.json`

这份文档是当前仓库的总体架构基线。
