# 接入飞书

这个文档用于说明如何把 OpenClaw 接入飞书，让客户能够直接在飞书里和小龙虾对话。

## 一、为什么建议接飞书

对大多数客户来说，飞书接入的意义很直接：

- 不用每次打开网页控制台
- 可以在手机上随时下任务
- 更适合企业内部协作
- 更适合把 OpenClaw 做成实际工作台

## 二、接入前准备

请先确认：

- OpenClaw 本体已正常安装
- `~/.openclaw` 已初始化
- 能执行 `openclaw --help`
- 能执行 `openclaw skills check`

## 三、安装飞书工具

执行：

```bash
npx -y @larksuite/openclaw-lark-tools install
```

## 四、按提示完成飞书机器人接入

安装后，通常会进入扫码或授权流程。

建议：

- 用手机飞书扫码
- 按提示创建机器人
- 完成后台审批

## 五、接入成功后的验证

验证点包括：

1. OpenClaw 控制台能看到 Feishu 频道
2. 飞书里能搜到机器人
3. 机器人能回复消息

## 六、推荐开启流式响应

为了提升体验，建议开启飞书流式回复：

```bash
openclaw config set channels.feishu.streaming true
```

如果改完配置后未生效，可重启网关：

```bash
openclaw gateway restart
```

如果你是做演示或试用，也可以直接执行仓库里的预设脚本：

```bash
./scripts/apply-demo-open-defaults.sh
```

## 七、常见问题

### 1. 机器人创建成功但无法回复

优先检查：

- 是否完成飞书后台审批
- 是否在 OpenClaw 控制台看到 Feishu 频道
- 网关是否正常运行

### 2. 流式响应不生效

优先检查：

- `channels.feishu.streaming` 是否已开启
- 是否已经重启网关

### 3. 多媒体消息发送失败

如果后续要让小龙虾发图片、音频、文件，建议：

- 把相关流程沉淀成 skill
- 或补到工作区记忆里

## 八、交付建议

给客户交付时，飞书接入应作为标准可选项，而不是默认强绑。

建议做法：

- 没有飞书需求的客户，不强行配置
- 有飞书需求的客户，再按本文档接入

## 九、一句话总结

飞书接入的价值，不是“多一个聊天入口”，而是把 OpenClaw 变成客户随时可用的工作助手。
