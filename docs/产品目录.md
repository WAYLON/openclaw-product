# 产品目录

这个文档用于展示当前仓库中已经存在的产品，以及未来可以扩展的产品类型。

## 一、产品分类

本仓库中的 OpenClaw 产品，按交付方式分为四类：

1. 基础版
2. 增强版
3. 客户专版
4. 团队版

## 二、当前已有产品

### 1. 通用股票研究包

路径：

- `stocks/openclaw-stock-pack/`

定位：

- 通用股票研究
- 适合选股、新闻判断、结构化研究路由

特点：

- 不带聚宽专用能力
- 更适合做基础版 / 引流版

### 2. LangLang 股票专用版

路径：

- `stocks/openclaw-stock-pack-langlang/`

定位：

- LangLang 专用股票研究包
- 包含聚宽 / JQData 研究能力

特点：

- 带客户专属工作区
- 带聚宽官方文档摘要
- 带 `jqdatasdk` 参考摘要
- 带最小验证脚本

这类产品属于：

- 客户专版

### 3. 教育产品骨架

路径：

- `education/openclaw-education-pack/`

定位：

- 教育场景产品预留骨架

特点：

- 当前不作为完整交付包
- 用于定义教育产品标准结构

## 四、推荐的产品矩阵

### 基础版

适合：

- 用于体验
- 用于低价引流
- 用于让客户先上手

### 增强版

适合：

- 增加行业能力
- 增加专业技能
- 增加工具接入

### 客户专版

适合：

- 根据客户习惯和流程定制
- 接客户数据源
- 接客户工具和工作流

### 团队版

适合：

- 多角色协作
- 多 Agent 配置
- 复杂工作流拆分

## 五、为什么要有产品目录

如果没有产品目录，项目会越来越像：

- 一堆文件
- 一堆技能
- 一堆临时客户修改

有了产品目录后，项目才会变成：

- 有产品线
- 有分层
- 有标准版和定制版
- 有未来扩展方向

## 六、一句话总结

产品目录的作用，不是列文件，而是告诉你：

**这个仓库将来卖什么、怎么卖、卖给谁。**
