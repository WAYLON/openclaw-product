# 默认运行规范

这个文档用于约定 OpenClaw 产品包在日常运行中的默认行为。

## 默认开启项

所有我们交付和内部使用的 OpenClaw 会话，默认开启：

- `/verbose`
- `/reasoning`

## 默认语言

所有我们交付和内部使用的 OpenClaw 会话，默认使用中文。

规则：

- 默认回答中文
- 默认说明中文
- 默认交付文本中文
- 只有命令、路径、环境变量、外部技能名、API 字段名保留原文

如果用户没有明确要求英文，就不要切到英文。

## 为什么默认开启

### 1. 方便排查问题

开启 `/verbose` 后，更容易看到工具调用和执行过程。

这对以下场景尤其重要：

- 首次部署
- 客户验收
- 技能调试
- 多工具协作
- 出现执行偏差时追踪原因

### 2. 方便理解 AI 在做什么

开启 `/reasoning` 后，更容易看到模型的推理过程或思考轨迹。

这对以下场景尤其重要：

- 股票研究结论解释
- 复杂任务拆解
- 验证是否真正理解业务规则
- 发现任务跑偏

## 交付要求

在给客户部署或初始化产品包时，应明确说明：

- 默认先开 `/verbose`
- 默认先开 `/reasoning`

如果是手动进入新会话，建议先执行：

```text
/verbose
/reasoning
```

## 例外情况

以下场景可以按需关闭：

- 明确需要省 Token
- 日常简单闲聊
- 已经验证稳定、只追求低成本运行

但默认交付和调试阶段，不建议关闭。
