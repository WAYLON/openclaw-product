# 模型配置与切换

这个文档用于说明 OpenClaw 中模型的配置、切换和基本使用建议。

## 一、为什么模型配置很重要

模型决定了：

- 小龙虾有多聪明
- 成本高不高
- 响应快不快
- 复杂任务是否稳定

所以模型配置不应该随便改，也不应该完全靠 AI 自己乱改。

## 二、推荐的配置方式

推荐优先使用 OpenClaw 自带的配置命令，而不是直接手改配置文件。

例如：

```bash
openclaw config
```

或者直接指定默认模型：

```bash
openclaw config set agents.defaults.model.primary "提供商/模型"
```

## 三、查看模型状态

执行：

```bash
openclaw models status
```

可以查看当前模型配置和状态。

## 四、临时切换模型

如果只是当前会话想切模型，建议使用斜杠命令：

```text
/model list
/model 提供商/模型
```

这种方式只影响当前会话，不改全局默认配置。

## 五、默认模型与降级模型

建议至少配置：

- 一个主力模型
- 一个降级模型

降级模型的作用是：

- 主模型欠费时兜底
- 服务异常时兜底
- 限流时兜底

## 六、建议的使用策略

### 简单任务

优先便宜、快的模型。

### 复杂任务

优先更强的模型。

### 子 Agent

优先配置更便宜的模型，控制成本。

## 七、不要怎么做

不建议：

- 直接手改配置文件然后不重启
- 让 AI 随意帮你改模型配置
- 不做验证就切到陌生模型

## 八、修改配置后建议执行

```bash
openclaw gateway restart
```

尤其是在你改的是全局配置而不是临时会话切换时。

## 九、交付建议

给客户交付时，模型配置要分两层：

1. 仓库只保留模板和说明
2. 客户自己在本地填真实模型配置

不要把真实 API Key 交付出去。

## 十、一句话总结

模型切换是 OpenClaw 使用中的高频动作，但配置应尽量标准化、模板化、可验证，不要靠记忆和手感。
