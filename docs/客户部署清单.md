# 客户部署清单

这个清单用于确认客户机器是否已经完成 OpenClaw 产品包的完整部署。

## 一、OpenClaw 本体

- [ ] 已安装 OpenClaw 本体
- [ ] `openclaw --help` 可执行
- [ ] 已生成 `~/.openclaw`
- [ ] 已存在 `~/.openclaw/openclaw.json`
- [ ] 已存在 `~/.openclaw/workspace/`
- [ ] 已存在 `~/.openclaw/skills/`

## 二、基础环境

- [ ] `git` 可执行
- [ ] `node` 可执行
- [ ] `npm` 可执行
- [ ] 已具备目标产品所需 Python 环境（如适用）

## 三、仓库与产品包

- [ ] 已克隆 `openclaw-product`
- [ ] 已确认要安装的行业目录
- [ ] 已确认要安装的产品包
- [ ] 已确认是 `skills-only` 还是 `full`

## 四、安装执行

- [ ] 已执行 `scripts/install-pack.sh`
- [ ] 安装过程中已备份旧配置（如适用）
- [ ] 安装完成后无报错

## 五、技能检查

- [ ] 已执行 `openclaw skills check`
- [ ] 目标产品包中的技能已被识别
- [ ] 目标产品包中的技能状态正常

## 六、本地密钥与账号

- [ ] 已填写模型或 provider 配置
- [ ] 已填写必要 API Key
- [ ] 已填写本地账号信息
- [ ] 已填写本机脚本路径
- [ ] 已填写终端或桥接方式

## 七、股票版额外检查

通用股票版：

- [ ] `stock-research-core` 可用
- [ ] `qveris-official` 可用

LangLang 股票版：

- [ ] `stock-research-core-langlang` 可用
- [ ] `jqdata-playbook` 可用
- [ ] 已确认 `jqdatasdk` 可导入
- [ ] 已确认聚宽认证可通过
- [ ] 已确认最小 `get_price` 可运行

## 八、工作区文件

如果采用 `full` 安装模式，再确认：

- [ ] `SOUL.md` 已安装
- [ ] `AGENTS.md` 已安装
- [ ] `TOOLS.md` 已安装
- [ ] `USER.md` 已安装
- [ ] `MEMORY.md` 已安装

## 九、最小验证

- [ ] 能回答“你现在是什么工作模式”
- [ ] 能说明当前使用的路由规则
- [ ] 能诚实指出缺失依赖
- [ ] 能完成一个最小业务示例

## 十、安全检查

- [ ] 未提交真实密钥
- [ ] 未将实例直接暴露到公网
- [ ] 未默认开启高危权限
- [ ] 已阅读 `docs/安全交付规范.md`

## 十一、交付完成标准

只有当下面三件事都满足时，才算真正交付完成：

1. 能安装
2. 能验证
3. 能安全使用
