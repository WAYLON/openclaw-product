# 接入微信

这个文档用于说明如何把 OpenClaw 接入微信个人号，以及在同一台机器上把第二个机器人 `bot-b` 单独接到微信。

## 一、先说结论

OpenClaw 的微信接入当前不是 core 内置频道，而是走官方文档收录的社区插件。

官方入口：

- OpenClaw 官方社区插件页：
  [Community plugins](https://docs.openclaw.ai/plugins/community)
- 微信插件仓库：
  [icesword0760/openclaw-wechat](https://github.com/icesword0760/openclaw-wechat)

官方文档里给出的安装命令是：

```bash
openclaw plugins install @icesword760/openclaw-wechat
```

## 二、整体架构

微信这条链路不是 OpenClaw 直接连微信，而是：

1. 微信个人号
2. WeChatPadPro
3. OpenClaw 微信插件
4. OpenClaw gateway
5. Agent / 模型

也就是：

`微信 <-> WeChatPadPro <-> openclaw-wechat <-> OpenClaw`

## 三、标准接入步骤

参考插件仓库说明，标准步骤是：

1. 安装 Node.js 22+
2. 安装 OpenClaw
3. 部署 WeChatPadPro
4. 生成 `TOKEN_KEY`
5. 安装微信插件
6. 在 OpenClaw 里写微信配置
7. 重启 gateway

插件文档里给出的关键命令包括：

```bash
openclaw plugins install @icesword760/openclaw-wechat
openclaw config set channels.wechat.enabled true
openclaw config set channels.wechat.serverUrl http://localhost:8849
openclaw config set channels.wechat.token YOUR_TOKEN_KEY
```

## 四、先装什么，后装什么

推荐顺序：

1. 先把 OpenClaw 本体装好
2. 先确认主 gateway 正常
3. 再装微信插件
4. 再配 WeChatPadPro 的地址和 token
5. 最后才测微信消息

不要一上来就：

- 先配微信
- 再补 gateway
- 再补模型

这样排错会很乱。

## 五、WeChatPadPro 这层要拿到什么

真正必须拿到的是两样：

1. `serverUrl`
例如：

```text
http://localhost:8849
```

2. `TOKEN_KEY`
插件仓库说明里给出的典型生成方式是：

```bash
curl "http://localhost:8849/login/GenAuthKey2?key=YOUR_ADMIN_KEY&count=1&days=365"
```

返回的 `authKeys` 值就是 OpenClaw 侧要填的 token。

## 六、主机器人怎么接微信

如果你接的是默认机器人，配置写到：

- `~/.openclaw/openclaw.json`

核心配置至少包括：

```json
{
  "channels": {
    "wechat": {
      "enabled": true,
      "serverUrl": "http://localhost:8849",
      "token": "YOUR_TOKEN_KEY"
    }
  }
}
```

然后重启主 gateway。

## 七、bot-b 怎么接微信

这是最容易说错的地方。

### 关键原则

`插件安装可以是全局一次，但 bot-b 的微信配置必须写进 bot-b 自己的 profile。`

也就是：

- 插件安装：全局装一次即可
- 配置位置：`~/.openclaw-bot-b/openclaw.json`
- gateway：`bot-b` 自己的端口，例如 `18799`

### bot-b 推荐结构

- 主机器人：
  - `~/.openclaw`
  - `18789`
- bot-b：
  - `~/.openclaw-bot-b`
  - `18799`

### bot-b 常用命令

这部分建议直接给交付人员或客户保存，后面排查基本都靠这些命令。

#### 1. 查看 bot-b 当前状态

```bash
openclaw --profile bot-b status
```

#### 2. 查看 bot-b 健康状态

```bash
openclaw --profile bot-b health --json
```

#### 3. 查看 bot-b 当前配置

```bash
cat /Users/waylon/.openclaw-bot-b/openclaw.json
```

如果只想看某一段配置：

```bash
openclaw --profile bot-b config get channels.wechat
openclaw --profile bot-b config get channels.feishu
```

#### 4. 修改 bot-b 配置

例如：

```bash
openclaw --profile bot-b config set channels.wechat.enabled true
openclaw --profile bot-b config set channels.wechat.serverUrl http://localhost:8849
openclaw --profile bot-b config set channels.wechat.token YOUR_TOKEN_KEY
```

#### 5. 安装 / 重装 bot-b 的 gateway

```bash
openclaw --profile bot-b gateway install --port 18799
```

#### 6. 重启 bot-b 的 gateway

```bash
launchctl kickstart -k gui/$(id -u)/ai.openclaw.bot-b
```

#### 7. 看 bot-b 的 gateway 日志

```bash
tail -f /Users/waylon/.openclaw-bot-b/logs/gateway.log
tail -f /Users/waylon/.openclaw-bot-b/logs/gateway.err.log
```

或者：

```bash
openclaw --profile bot-b logs --follow
```

#### 8. 本地直接和 bot-b 对话

```bash
openclaw --profile bot-b tui
```

或者最小验证：

```bash
openclaw --profile bot-b tui --session botb-test --message "回复 BOTB_OK" --timeout-ms 20000
```

#### 9. 查看 bot-b 端口是否正常

```bash
lsof -nP -iTCP:18799 -sTCP:LISTEN
```

### bot-b 配置示例

```json
{
  "channels": {
    "wechat": {
      "enabled": true,
      "serverUrl": "http://localhost:8849",
      "token": "YOUR_TOKEN_KEY"
    }
  }
}
```

注意：

- 这里的 `token` 是 WeChatPadPro 给的 token
- 不是 OpenClaw gateway token
- 也不是飞书的 App Secret

### bot-b 的正确接法

1. 先确认 `bot-b` 的 gateway 已独立跑起来
2. 再把微信配置写到 `~/.openclaw-bot-b/openclaw.json`
3. 再重启 `bot-b` 的 gateway
4. 最后用微信给 `bot-b` 触发消息

不要把微信配置写回：

- `~/.openclaw/openclaw.json`

否则最后常见结果就是：

- 你以为接的是 `bot-b`
- 实际响应的是主机器人

## 八、推荐的微信触发策略

插件仓库里给了几个很实用的配置项：

- `triggerPrefix`
- `replyPrefix`
- `dmPolicy`
- `allowFrom`
- `groupPolicy`

如果这台微信号本身还要正常做人用，推荐一开始就加：

```json
{
  "channels": {
    "wechat": {
      "triggerPrefix": "@ai"
    }
  }
}
```

这样不会把所有日常聊天都喂给机器人。

## 九、双机器人场景怎么讲给客户

最稳的说法是：

- 飞书和微信都可以接
- 但“第二个机器人”不是简单复制一个聊天入口
- 必须是独立 profile、独立配置、独立 gateway

对于 `bot-b` 接微信，建议直接这样讲：

1. 微信插件装一次就够
2. `bot-b` 只需要单独写自己的微信配置
3. `bot-b` 的消息和主机器人消息不要混用同一套状态目录
4. `bot-b` 是否回复，要看 `bot-b` 自己的日志，不要看主 gateway

## 十、这次实战踩出来的经验

这次双机器人实战后，微信这边建议固定遵守：

1. 先确认主 gateway 和 `bot-b` gateway 已彻底分开
2. 再接第二个渠道
3. 先确认消息有没有进入对应 profile 的日志
4. 再去查模型是否回复
5. 渠道 token、模型 token、gateway token 三者不能混

### 本次 `bot-b` 实测成功口径

本次已经在 `bot-b` profile 下完成了一次真实微信接入，确认点包括：

1. 安装命令要明确绑定到 `bot-b`：

```bash
OPENCLAW_STATE_DIR=/Users/waylon/.openclaw-bot-b \
OPENCLAW_CONFIG_PATH=/Users/waylon/.openclaw-bot-b/openclaw.json \
npx -y @tencent-weixin/openclaw-weixin-cli@latest install
```

2. 安装器扫码成功后，会把账号状态写进：

- `~/.openclaw-bot-b/openclaw-weixin/accounts/`

3. 安装器结束时不一定会正确重启 `bot-b` 自己的 gateway`

所以建议手动补一次：

```bash
launchctl kickstart -k gui/$(id -u)/ai.openclaw.bot-b
```

4. 最终验收不能只看“插件安装成功”，而要看：

```bash
openclaw --profile bot-b status
```

理想状态应该出现：

- `openclaw-weixin | ON | OK`
- `accounts 1/1`

如果还没有这两项，就不能算接入完成。

## 十一、一句话总结

微信接入的重点，不是“把插件装上”，而是把：

- WeChatPadPro
- 插件
- profile
- gateway

这四层对应关系理顺。
