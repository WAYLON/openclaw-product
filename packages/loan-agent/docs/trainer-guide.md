# 助贷 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲流程定位
2. 再演示材料清单和预检
3. 最后讲合规边界

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的灵魂、提示词、技能绑定、记忆与文档。
- 它主要解决的是：线索采集、资格预检、材料引导、预约与合规流程交付。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请先做资格预检并列出还缺哪些材料。
- 请把这份收入证明解析成中文摘要。
- 请生成一份面向客户的合规材料清单说明。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 合规风险
  - 判断依据
    - 材料完整度
    - 基础条件
    - 流程约束
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只讲流程，而是会先判断当前该继续推进、先补材料，还是暂不推进。

## 首次演示推荐案例
- 输入：
  - 客户做餐饮经营 2 年，有流水但收入证明不完整，请判断现在适合继续推进、先补材料还是暂不推进。
- 输出：
  - 输出一份结构化判断结果，包括结论、合规风险、材料完整度/基础条件/流程约束依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它能直接体现助贷 Agent 的核心价值：不是只列材料，而是先判断现在能不能往下走、该怎么走。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入客户情况，输出“继续推进 / 先补材料 / 暂不推进”的判断结果。
  - 输入客户场景，输出可直接发给客户的材料说明和合规话术。
- 需先说明“按接入情况启用”的能力：
  - 接入 OCR 服务后，可补充身份证、收入证明、流水等材料解析。
  - 接入表单、CRM、预约、签署平台后，可补充留资、记录、预约和签署动作。
- 当前不应默认承诺的能力：
  - 未接入 OCR 或业务平台时，不会默认演示自动识别材料或全流程自动流转。
  - 不替代机构做正式审批、授信或放款承诺。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 合规风险 + 判断依据 + 下一步建议”的流程判断输出，让顾问和客户都知道现在该先做什么。

## 1分钟演示版本
- 推荐提问：
  - 我这种情况现在是继续推进，还是先补材料？
- 预期输出结构：
  - 会直接输出结论、合规风险、材料完整度/基础条件/流程约束依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会列材料，而是会先判断现在该不该往下走。
  - 客户看到的是“当前能不能推进”和“先补什么”的明确结果。

## 重点风险边界
- 不替代授信审批
- 不承诺放款结果
- 不建议规避合规要求
