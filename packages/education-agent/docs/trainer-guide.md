# 教育 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲教学设计能力
2. 再演示课堂资料解析
3. 最后演示语音转写或播报

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的官方核心文件、技能绑定、记忆与文档。
- 它主要解决的是：备课、题目、讲义解析、语音课堂接入与老师讲解交付。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请设计一节函数图像课的大纲。
- 请把这段课堂录音整理成中文纪要。
- 请把这份讲义改成老师可播报的语音串讲稿。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 优先级
  - 判断依据
    - 学习目标
    - 学生阶段
    - 课堂资源
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只给教学素材，而是会先判断这节课更适合讲授、练习还是互动，再给出下一步安排。

## 首次演示推荐案例
- 输入：
  - 请判断“初二函数图像入门”这节课更适合先讲授、先练习还是先互动，并给出依据和下一步建议。
- 输出：
  - 输出一份结构化判断结果，包括结论、优先级、学习目标/学生阶段/课堂资源依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它能让客户立刻看懂，这个 Agent 不只是会出教案，而是会帮老师判断这节课该怎么上。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入学科、学段和课堂目标，输出“先讲什么、先练什么、先互动什么”的判断结果。
  - 输入讲义主题，输出老师可直接口头使用的课堂提纲和讲解说明。
- 需先说明“按接入情况启用”的能力：
  - 接入语音服务后，可补充课堂录音转写、语音播报和口语练习。
  - 接入题库或 LMS 后，可补充测验运行、课程管理和学习进度协同。
- 当前不应默认承诺的能力：
  - 未接入语音服务时，不会默认演示录音转写或语音播报结果。
  - 未接入题库或 LMS 时，不会默认演示实时课堂平台或课程系统联动。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 优先级 + 判断依据 + 下一步建议”的教学判断输出，老师可以直接拿来决定这节课怎么安排。

## 1分钟演示版本
- 推荐提问：
  - 这节课我应该先讲、先练，还是先互动？
- 预期输出结构：
  - 会直接输出结论、优先级、学习目标/学生阶段/课堂资源依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会出教案，而是先判断这节课最合适的上法。
  - 客户看到的是“课堂决策结果”，不是一段泛泛教学建议。

## 重点风险边界
- 不替代正式评分
- 不把语音误识别内容当事实
- 不替代老师做高风险学生判断
