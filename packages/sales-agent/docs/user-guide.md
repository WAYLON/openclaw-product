# 客服 / 销售转化 Agent 用户指南

## 1. 用户能获得什么
- 当前可直接使用：中文客服回复建议、线索判断建议、流程说明
- 按接入情况可启用：工单分流、CRM 跟进、预约与合同确认

## 1.1 当前交付状态说明
- 已内置：回复口径、会话总结、转化建议
- 需外部配置：部分记录同步或增强型流程能力
- 需第三方平台：客服台、CRM、问卷、预约、签署等平台型能力
- 仅规划：未单独交付的导出链路不作为默认开箱承诺

## 1.2 判断结果输出格式
- 结论：值得推进 / 继续观察 / 先不推进
- 转化概率：高 / 中 / 低
- 判断依据：
  - 意向强度：是否主动问价格、方案、交付时间，还是只泛泛咨询
  - 当前阶段：处于初次咨询、比价、异议处理、临门一脚还是投诉阶段
  - 下一步动作判断：当前更适合回复、预约、补资料、转人工还是先停一停
- 说明：用简短中文说明为什么这样判断
- 下一步建议：继续跟进、发客户话术、安排预约，或先稳住服务

## 2. 正确提问方式
- 先说你的目标。
- 再说你现有资料。
- 再说明你希望输出什么格式。
- 如果涉及外部平台，请说明是否已经完成初始化。

## 3. 常见使用流程
1. 确认当前会话类型
2. 补齐客服/CRM/预约相关 Key
3. 演示回复建议
4. 演示跟进记录

## 4. 什么时候需要先初始化 Key
- 只有在你要启用客服台、CRM、问卷或预约等平台能力时，才需要初始化对应凭证。
- 当前交付若仅做中文回复建议、线索判断和会话总结，可先不配置这些外部 Key。

## 5. 风险与边界
- 不能夸大承诺
- 不能违规外呼
- 不能把投诉场景硬推成交
