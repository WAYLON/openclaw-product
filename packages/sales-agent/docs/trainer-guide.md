# 客服 / 销售转化 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲服务和销售一体化
2. 再演示回复与跟进
3. 最后讲投诉边界

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的灵魂、提示词、技能绑定、记忆与文档。
- 它主要解决的是：FAQ、线索识别、跟进推进、预约转化与记录沉淀。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请判断这段咨询属于 FAQ、销售机会还是投诉。
- 请给一版客户回复和一版内部 CRM 跟进建议。
- 请判断客户所处销售阶段并给出下一步动作。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 转化概率
  - 判断依据
    - 意向强度
    - 当前阶段
    - 推进阻力
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只给一句客服回复，而是会先判断值不值得推进、卡在什么地方、下一步该怎么推。

## 首次演示推荐案例
- 输入：
  - 客户反复问价格和交付时间，但一直不确认是否安排演示，请判断现在值不值得推进，并给出转化概率、依据和下一步建议。
- 输出：
  - 输出一份结构化判断结果，包括结论、转化概率、意向强度/当前阶段/推进阻力依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它最能体现销售 Agent 的差异：不是只会回复，而是会判断成交可能性和推进卡点。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入客户对话，输出“值得推进 / 继续观察 / 先不推进”的判断结果。
  - 输入客户场景，输出可直接发给客户的回复话术和内部跟进建议。
- 需先说明“按接入情况启用”的能力：
  - 接入客服台、CRM、问卷、预约或签署平台后，可补充记录、预约和流程联动。
  - 接入外部业务平台后，可补充工单分发和客户档案同步。
- 当前不应默认承诺的能力：
  - 未接入外部平台时，不会默认演示 CRM 自动更新、自动外呼或全链路自动化。
  - 不会把投诉场景硬推成销售机会，也不会承诺自动成交。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 转化概率 + 判断依据 + 下一步建议”的销售判断输出，帮助团队知道这单到底该不该继续推。

## 1分钟演示版本
- 推荐提问：
  - 这个客户现在值得继续跟吗？
- 预期输出结构：
  - 会直接输出结论、转化概率、意向强度/当前阶段/推进阻力依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会写一句回复，而是会判断这单有没有机会、卡点在哪、下一步怎么推。
  - 客户看到的是“推进判断”，不是简单客服话术。

## 重点风险边界
- 不能夸大承诺
- 不能违规外呼
- 不能把投诉场景硬推成交
