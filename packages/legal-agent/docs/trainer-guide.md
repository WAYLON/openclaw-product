# 法务 / 合同流程 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲合同流程而不是法律替代
2. 再演示解析和比对
3. 最后讲签署状态与留痕

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的灵魂、提示词、技能绑定、记忆与文档。
- 它主要解决的是：合同解析、条款比对、审批流转、签署跟踪与证据打包。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请解析这份合同并标出重点复核条款。
- 请比对两个版本合同。
- 请生成一份签署流程说明。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 风险等级
  - 判断依据
    - 条款风险
    - 合规边界
    - 介入级别
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只复述合同内容，而是会先判断是否需要重点复核、是否超出流程助手边界、是否应请律师介入。

## 首次演示推荐案例
- 输入：
  - 请判断这份合同中的付款条款和违约责任是否需要重点复核，并给出风险等级、依据和下一步建议。
- 输出：
  - 输出一份结构化判断结果，包括结论、风险等级、条款风险/合规边界/介入级别依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它能让客户马上看懂，这个 Agent 不只是会看文档，而是会提示边界、风险和升级处理动作。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入条款问题，输出“可继续推进 / 需重点复核 / 建议律师介入”的判断结果。
  - 输入流程场景，输出可直接转发的风险提示或流程说明。
- 需先说明“按接入情况启用”的能力：
  - 接入 OCR、审批表单、签署或状态平台后，可补充识别、发起、跟踪和留痕动作。
  - 接入外部业务平台后，可补充流程联动和状态回写。
- 当前不应默认承诺的能力：
  - 未接入 OCR 或签署平台时，不会默认演示扫描件解析、自动发起签署或状态同步。
  - 不替代律师出具正式法律意见，也不会自动确认法律结论。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 风险等级 + 判断依据 + 下一步建议”的法务判断输出，帮助团队知道该复核、该升级，还是该请律师介入。

## 1分钟演示版本
- 推荐提问：
  - 这份合同现在能继续推进，还是需要重点复核？
- 预期输出结构：
  - 会直接输出结论、风险等级、条款风险/合规边界/介入级别依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会讲合同内容，而是会先判断能不能继续推进、风险在哪、要不要升级处理。
  - 客户看到的是“边界清楚的法务判断”，不是替代律师的随意意见。

## 重点风险边界
- 不替代律师正式意见
- 不能自动签署未经确认文件
- 不能把 OCR 抽取当法律定性
