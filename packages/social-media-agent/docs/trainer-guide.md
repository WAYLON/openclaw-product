# 社交媒体 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道当前能力边界，不把未接入的链路讲成已上线。

## 推荐讲解顺序
1. 先讲它不是发帖机器人，而是社交内容整理与辅助分析 Agent
2. 再演示热点浏览、内容抽取和摘要整理
3. 最后讲哪些动作需要后续再接平台账号或发布链路

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它当前基于共享技能工作，重点能力是网页访问、内容浏览、摘要和信息整理。
- 它适合做热点跟踪、素材整理、评论区观察和内容准备。
- 当前没有默认承诺自动发帖、自动群发或账号托管。

## 标准演示脚本
- 请整理今天和我们品牌相关的三条热点内容。
- 请提取这篇社交平台内容的核心观点并生成摘要。
- 请把这组素材整理成适合后续改写的内容提纲。

## 首次演示推荐案例
- 输入：
  - 请浏览一条公开网页内容，提取重点信息，并整理成适合二次创作的提纲。
- 输出：
  - 输出一份结构化结果，包括主题、重点观点、适合保留的表达、风险点和后续可加工方向。
- 为什么这个案例适合第一次演示：
  - 它能直接体现这个 Agent 当前真实已接入的共享技能能力，不会误导客户以为已经接好了自动发布。

## 承诺边界提醒
- 可默认演示的能力：
  - 浏览网页内容并提取信息。
  - 对文章、帖子或页面内容做摘要与结构整理。
  - 输出适合后续改写、排版或分发的内容框架。
- 需先说明“按接入情况启用”的能力：
  - 接入具体社交平台账号后，可补充平台内读取、搜索或互动动作。
  - 接入发布链路后，可补充自动发布或排程分发。
- 当前不应默认承诺的能力：
  - 不默认承诺自动发帖。
  - 不默认承诺账号托管。
  - 不默认承诺跨平台自动同步发布。

## 1分钟演示版本
- 推荐提问：
  - 请把这篇网页内容整理成一版适合社交媒体改写的提纲。
- 预期输出结构：
  - 主题
  - 重点观点
  - 可复用表述
  - 风险提醒
  - 下一步建议
- 讲解话术：
  - 当前它最适合做前置内容整理，不是直接替团队接管所有社交平台动作。

## 重点风险边界
- 不把来源不明内容当成已核实事实。
- 不默认发布高风险内容。
- 不默认接管客户账号密码或长期托管。
