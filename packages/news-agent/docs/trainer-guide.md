# 资讯 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲资讯不是搜索而是归并
2. 再演示日报生成
3. 最后讲发布边界

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的技能绑定、记忆与文档；工作区核心文件由官方生成。
- 它主要解决的是：多源资讯归并、摘要、日报周报与发布准备。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请整理今天 AI 行业晨报。
- 请监控三个关键词并合并重复内容。
- 请生成一份竞品周报框架。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 信息可信度
  - 判断依据
    - 来源可靠性
    - 信息时效性
    - 重复与噪声
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只堆资讯，而是会先判断哪些值得保留、哪些只需关注、哪些暂不建议直接引用。

## 首次演示推荐案例
- 输入：
  - 请判断这 3 条 AI 资讯哪些值得写进今天的日报，并给出可信度、依据和下一步建议。
- 输出：
  - 输出一份结构化判断结果，包括结论、信息可信度、来源可靠性/信息时效性/重复与噪声依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它最容易让客户看见，这个 Agent 不只是会摘要，而是会帮助决定哪些信息该留下来。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入多条资讯素材，输出“保留 / 继续关注 / 暂缓引用”的判断结果。
  - 输入主题，输出一版可直接扩写的日报或周报结构说明。
- 需先说明“按接入情况启用”的能力：
  - 接入 RSS、搜索或内容平台后，可补充订阅源拉取、联网增强和发布链路。
  - 接入 CMS 后，可补充发布与知识库同步动作。
- 当前不应默认承诺的能力：
  - 未接入 RSS 或搜索服务时，不会默认抓取外部实时资讯。
  - 未接入内容平台时，不会默认演示自动发布或全链路同步。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 信息可信度 + 判断依据 + 下一步建议”的资讯判断输出，帮助团队决定哪些信息值得用。

## 1分钟演示版本
- 推荐提问：
  - 这几条资讯里，哪些值得写进今天的简报？
- 预期输出结构：
  - 会直接输出结论、信息可信度、来源可靠性/信息时效性/重复与噪声依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会总结新闻，而是会判断哪些信息值得留下来。
  - 客户看到的是“可直接用于简报的取舍结果”，不是一堆资讯堆叠。

## 重点风险边界
- 不能把传闻当已验证事实
- 不能跳过审核直接发布高风险内容
