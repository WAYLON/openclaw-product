# 像素修图师 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲预检思路
2. 再演示一张图的处理
3. 最后讲环境差异和边界

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的灵魂、提示词、技能绑定、记忆与文档。
- 它主要解决的是：修图、超分、背景移除、老照片修复与交付说明。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请先预检这张老照片适合哪些修复。
- 请把这批商品图做背景移除并标记需人工复查项。
- 请说明 CPU 和 GPU 两种方案区别。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 预期效果
  - 判断依据
    - 原图质量
    - 信息保留度
    - 交付要求
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只说“能不能修”，而是会明确判断值不值得修、效果能到什么程度、哪些地方修不回来。

## 首次演示推荐案例
- 输入：
  - 这张老照片人脸有点糊、背景发黄、分辨率也不高，请判断值得修、只建议增强还是建议人工处理。
- 输出：
  - 输出一份结构化判断结果，包括结论、预期效果、原图质量/信息保留度/交付要求依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它能让客户立刻看懂，Agent 不只是接单，而是先判断效果预期和不可修复点。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入图片问题描述，输出“值得修 / 只建议增强 / 建议人工处理”的判断结果。
  - 输入交付场景，输出可直接发给客户的修图预期说明和交付话术。
- 需先说明“按接入情况启用”的能力：
  - 接入修图引擎、超分或背景移除服务后，可补充真实图像处理结果。
  - 接入批量处理链路后，可补充批量增强或批量抠图。
- 当前不应默认承诺的能力：
  - 未接入图像服务时，不会默认演示真实修图结果回传。
  - 不会承诺所有老照片、人像或商品图都能被完全修复。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 预期效果 + 判断依据 + 下一步建议”的修图判断输出，先让客户知道值不值得做。

## 1分钟演示版本
- 推荐提问：
  - 这张图值得修吗，还是只建议简单增强？
- 预期输出结构：
  - 会直接输出结论、预期效果、原图质量/信息保留度/交付要求依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会说“能修”，而是会先判断值不值得做、能做到什么程度。
  - 客户看到的是“效果预期和限制”，不是一句模糊答复。

## 重点风险边界
- 不承诺无限恢复
- 不伪造证据图片
- 不把猜测补全当真实修复
