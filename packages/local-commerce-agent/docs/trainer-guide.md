# 电商 / 本地生活 Agent 讲师讲解手册

> 这是正式交付物，面向老师、实施顾问、售前顾问。

## 讲解目标
- 让客户理解这个 Agent 解决什么问题。
- 让客户知道首次怎么装、怎么配、怎么测。
- 让客户知道风险边界与最佳实践。

## 推荐讲解顺序
1. 先讲经营场景
2. 再演示商品和活动
3. 最后讲预约和复盘

## 讲师话术模板
- 这不是一个总控机器人，而是一个独立专业 Agent。
- 它有自己的灵魂、提示词、技能绑定、记忆与文档。
- 它主要解决的是：商品运营、活动内容、客户标签、预约到店与经营复盘。
- 如果第一次调用外部能力缺少 Key，它不会直接报错，而会先提示你如何初始化。

## 标准演示脚本
- 请写 3 版活动宣传文案。
- 请设计一个预约到店跟进流程。
- 请把这组商品图处理后给出详情页说明文案。

## 演示时的标准输出结构
- 讲师演示时，建议固定展示以下结构：
  - 结论
  - 转化优先级
  - 判断依据
    - 用户意图
    - 场景适配
    - 资源条件
  - 说明
  - 下一步建议
- 建议讲师提醒客户：这里不是只给一段文案，而是会先判断“先推什么、先讲什么、先约什么”。

## 首次演示推荐案例
- 输入：
  - 客户问“现在有什么优惠，今天能不能直接到店”，请判断先讲活动、先讲商品还是先约到店。
- 输出：
  - 输出一份结构化判断结果，包括结论、转化优先级、用户意图/场景适配/资源条件依据、说明和下一步建议。
- 为什么这个案例最适合第一次演示：
  - 它能让客户马上看懂，这个 Agent 不只是会写活动文案，而是会帮门店判断怎么接待更容易成交。

## 承诺边界提醒
- 可默认演示的能力：
  - 输入门店或活动场景，输出“先推活动 / 先讲商品 / 先约到店”的判断结果。
  - 输入门店信息，输出可直接使用的活动文案、FAQ 口径和到店说明。
- 需先说明“按接入情况启用”的能力：
  - 接入商品目录、CRM、客服台、预约系统后，可补充目录查询、客户记录和预约承接。
  - 接入图像处理服务后，可补充商品图清理或背景移除。
- 当前不应默认承诺的能力：
  - 未接入业务平台时，不会默认演示完整商品后台、客服台或预约系统联动。
  - 未接入图像服务时，不会默认演示图片增强和批量图像处理。

## 讲解话术建议
- 讲师可直接这样说：
  - 这不是普通聊天，而是“结论 + 转化优先级 + 判断依据 + 下一步建议”的经营判断输出，帮助门店先做最容易成交的动作。

## 1分钟演示版本
- 推荐提问：
  - 这个客户现在应该先讲优惠，还是先约到店？
- 预期输出结构：
  - 会直接输出结论、转化优先级、用户意图/场景适配/资源条件依据，以及下一步建议。
- 讲解话术：
  - 这不是普通聊天，它不是只会写活动文案，而是会判断先做哪个动作更容易成交。
  - 客户看到的是“先推什么”的经营判断，不是空泛的营销建议。

## 重点风险边界
- 不能做虚假优惠承诺
- 不能夸大库存和权益
- 不能跳过平台规则
