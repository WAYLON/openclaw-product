# 电商 / 本地生活 Agent 用户指南

## 1. 用户能获得什么
- 当前可直接使用：中文活动文案、FAQ 回答、预约流程说明
- 按接入情况可启用：商品目录查询、客户标签、经营复盘、商品图处理

## 1.1 当前交付状态说明
- 已内置：文案、问答、流程说明、基础经营口径
- 需外部配置：部分图片处理或经营数据整理能力
- 需第三方平台：商品目录、CRM、客服台、预约系统、自动化工作流
- 仅规划：未单独交付的报表导出链路不作为默认开箱承诺

## 1.2 判断结果输出格式
- 结论：先推活动 / 先讲商品 / 先约到店
- 转化优先级：高 / 中 / 低
- 判断依据：
  - 用户意图：是在问价格、问活动、问商品细节，还是问到店流程
  - 场景适配：更适合线上成交、线下体验、拉新还是复购
  - 资源条件：当前是否有库存、优惠、门店承接能力或预约余量
- 说明：用简短中文说明为什么先推这个动作
- 下一步建议：先发文案、先解释商品、先引导预约，或先补经营信息

## 2. 正确提问方式
- 先说你的目标。
- 再说你现有资料。
- 再说明你希望输出什么格式。
- 如果涉及外部平台，请说明是否已经完成初始化。

## 3. 常见使用流程
1. 确认门店或商品场景
2. 补齐内容/CRM/预约相关 Key
3. 演示文案与预约
4. 再讲复盘指标

## 4. 什么时候需要先初始化 Key
- 只有在你要启用内容后台、CRM、客服台、预约或自动化工作流时，才需要初始化对应凭证。
- 当前交付若仅做中文活动文案、FAQ 和流程说明，可先不配置这些外部 Key。

## 5. 风险与边界
- 不能做虚假优惠承诺
- 不能夸大库存和权益
- 不能跳过平台规则
