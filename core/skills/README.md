# Core 基础技能层

这个目录不是行业技能目录，而是所有产品包共用的基础技能预设层。

它解决三个问题：

1. 先给客户一套通用的 OpenClaw 基础能力
2. 再在行业目录里叠加股票、教育等领域能力
3. 交付时不再把“基础技能”和“行业技能”混在一起

## 当前默认基础技能

当前基础层采用一套固定的 10 技能预设，清单见：

- `basic-10.lock.json`

这 10 个技能的定位是：

- 联网搜索
- 技能发现
- 主动巡检
- 浏览器自动化
- 视频生成
- 定时任务
- 长任务规划
- 内容创作工具包
- 官方技能库
- 国产图像生成

## 为什么这里不直接放完整第三方源码

这里的很多技能来自外部开源仓库或官方商店，真正的交付重点不是把源码散着复制进仓库，而是：

- 锁定默认技能名单
- 固化安装命令
- 固化检查脚本
- 固化交付规范

因此，这个目录里保存的是：

- 技能锁定清单
- 安装约定
- 校验脚本

实际安装入口在：

- `../scripts/install-core-skills.sh`
- `../scripts/check-core-skills.sh`

## 与行业包的关系

关系如下：

1. `core/skills/`
   - 通用基础技能层
2. `stocks/*/skills/`
   - 股票行业技能层
3. `education/*/skills/`
   - 教育行业技能层

也就是说：

- `stock-research-core`
- `qveris-official`

这些都不再算“基础技能”，而是股票行业附加技能。
