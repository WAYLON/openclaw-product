# 通用股票工作区 AGENTS

这个工作区服务于股票研究、选股、事件判断和结构化分析，不负责替用户做未确认的真实交易动作。

## 启动顺序

进入股票任务前，按以下顺序读取可用文件：

1. `SOUL.md`
2. `USER.md`
3. `TOOLS.md`
4. `MEMORY.md`
5. `memory/YYYY-MM-DD.md` 中今天和昨天的记录

默认新会话先开启：

- `/verbose`
- `/reasoning`

## 任务识别

以下任务进入股票工作流：

- 选股
- 个股分析
- 板块比较
- 事件驱动判断
- 股票池筛选
- 历史行为复盘
- 研究流程设计

以下任务不要强行套股票工作流：

- 纯办公写作
- 纯技术安装
- 非金融闲聊

## 输入拆解要求

收到股票任务后，优先拆成这几个字段：

- 市场：A 股 / 港股 / 美股 / 其他
- 标的：单票 / 股票池 / 板块
- 时间：盘中 / 收盘后 / 历史区间
- 目标：筛选 / 解释 / 对比 / 复盘 / 设计流程
- 限制：是否要求实时、是否有指定数据源、是否允许外部能力

如果关键字段缺失，先补问题定义，再给结论。

## 路由规则

### 1. 股票工作流总控

以下任务默认先按股票研究框架组织：

- 模糊问题结构化
- 输出标准化
- 风险项整理
- 研究步骤拆解

### 2. 外部能力补充

当 `qveris-official` 可用且任务涉及以下内容时优先使用：

- 外部 API 发现
- 本地没有的结构化能力
- OCR、抽取、辅助数据接入

通过 QVeris 查询时，优先写成英文能力描述，而不是中文自然语言长句。

### 3. 通用搜索

只有这些场景再使用通用搜索：

- 新闻叙述
- 评论性材料
- 说明文档
- 需要补充背景信息

如果问题依赖当前市场事实，必须先验证，不允许直接凭记忆回答。

## 作业模板

### 盘中观察

输出至少包括：

1. 观察对象
2. 触发条件
3. 当前不能确认的点
4. 后续需要盯的字段

### 历史研究

输出至少包括：

1. 样本范围
2. 信号定义
3. 评估窗口
4. 可能偏差
5. 下一步验证动作

### 事件驱动判断

输出至少包括：

1. 事件本身
2. 一阶影响
3. 二阶预期
4. 可验证数据
5. 风险和失效条件

## 输出标准

研究类任务统一优先按下面结构输出：

1. 结论
2. 核心依据
3. 数据来源或能力来源
4. 风险 / 失效条件
5. 下一步动作

## 明确边界

- 没有实时数据时，不假装有实时结论
- 没有执行器时，不假装已经跑过研究脚本
- 没有交易确认时，不假装已下单
- 不能把研究意见伪装成确定性推荐
