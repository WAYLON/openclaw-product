# 股票研究总控技能

当用户在讨论股票研究，而不是普通聊天时，使用这个技能。

## 主要目标

- 把模糊的股票问题改写成结构化研究任务
- 把任务路由到合适的能力
- 让输出更稳定、更适合决策

## 路由规则

### 当 `jqdata-research` 可用时，以下任务优先使用它：

- A 股历史 K 线研究
- 分钟级或日级研究
- 因子式筛选
- 可复用选股流程
- 回测与历史验证

### 当 `qveris-official` 可用时，以下任务优先使用它：

- 发现新的外部 API 或能力
- 处理本地技能没有覆盖的数据领域
- OCR、抽取、特殊外部工具
- 盘中选股、盘中筛选、实时行情类任务优先通过 `https://qveris.ai/` 提供的同花顺工具能力完成

使用 QVeris 做 discover 时，查询必须写成英文能力描述。

### 以下任务可用通用搜索：

- 偏定性的新闻解读
- 评论性内容
- 文档和教程
- 以叙述性资料为主、不是结构化数据的任务

## 研究检查清单

回答股票问题前，至少检查：

1. 用户要的是当前事实，还是历史研究？
2. 市场是 A 股、港股、美股还是宏观？
3. 答案需要结构化数据、叙述性材料，还是两者都要？
4. 这是选股、回测、逻辑评估，还是新闻判断？
5. 结论的失效条件是什么？

## 输出模板

### 个股判断

1. 结论
2. 证据
3. 数据来源
4. 风险
5. 下一步观察

### 选股结果

1. 信号定义
2. 股票池
3. 时间窗口
4. 排序方法
5. 候选列表
6. 注意事项

### 回测结果

1. 假设
2. 数据与样本期
3. 进出场逻辑
4. 成本与约束
5. 结果
6. 局限

## 非目标

- 这个技能本身不直接提供市场数据
- 这个技能不能被用来编造实时市场事实
- 这个技能不应在非股票任务中覆盖用户原有的助手人格
