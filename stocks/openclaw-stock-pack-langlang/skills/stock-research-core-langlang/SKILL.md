# LangLang 股票研究总控技能

当用户在讨论 LangLang 场景下的股票研究，而不是普通聊天时，使用这个技能。

## 主要目标

- 把结果条件改写成结构化研究问题
- 用 JQData 做前置共性倒推
- 支持三条研究链路并行分析
- 先输出共性和结论，再生成 JoinQuant / JQData 回测代码

## 路由规则

### 当 `jqdata-research` 可用时，以下任务优先使用它：

- A 股历史 K 线研究
- 分钟级或日级研究
- 因子式筛选
- 可复用选股流程
- 回测与历史验证
- 结果条件反推前置共性

### LangLang 研究默认顺序

1. 把用户给的结果条件转成研究问题
2. 并行开 3 条研究链路
3. 汇总重合结论和冲突项
4. 输出共性和研究结论
5. 生成 JoinQuant / JQData 回测代码
6. 由用户自己粘贴到聚宽回测

三条研究链路优先建议为：

- 不同时间窗口
- 不同过滤条件
- 不同市场环境分组

### 当 `qveris-official` 可用时，以下任务优先使用它：

- 发现新的外部 API 或能力
- 处理本地技能没有覆盖的数据领域
- OCR、抽取、特殊外部工具
- 策略被认可后的盘中选股、盘中筛选、实时行情类任务优先通过 `https://qveris.ai/` 提供的同花顺工具能力完成

使用 QVeris 做 discover 时，查询必须写成英文能力描述。

### 以下任务可用通用搜索：

- 偏定性的新闻解读
- 评论性内容
- 文档和教程
- 以叙述性资料为主、不是结构化数据的任务

## 研究检查清单

回答股票问题前，至少检查：

1. 用户要的是当前事实，还是历史研究？
2. 市场是 A 股、港股、美股还是宏观？
3. 答案需要结构化数据、叙述性材料，还是两者都要？
4. 这是选股、回测、逻辑评估，还是新闻判断？
5. 结论的失效条件是什么？

## 输出模板

### 共性研究结果

1. 研究问题
2. 三条研究链路的核心发现
3. 交叉验证后的共性
4. 风险和冲突项
5. 下一步验证

### 回测代码输出

1. 研究问题
2. 共性定义
3. 信号定义
4. 回测代码
5. 风险与局限

## 非目标

- 这个技能本身不直接提供市场数据
- 这个技能不能被用来编造实时市场事实
- 这个技能不应在非股票任务中覆盖用户原有的助手人格
- 这个技能不是盘中执行器
