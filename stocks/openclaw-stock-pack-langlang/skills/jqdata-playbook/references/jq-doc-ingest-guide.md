# 聚宽文档整理指引

当你准备把更多聚宽文档整理进这个技能时，按这份规则处理。

## 应该保留什么

- 你会反复用到的精简 API 说明
- 会影响研究设计的字段定义
- 正确数据语义的示例
- 已知数据限制

## 不应该保留什么

- 没有整理过的大段原始厂商手册
- 已经在别处重复记录的内容
- 账号、密码或密钥

## 推荐组织方式

- 每个主题一个参考文件
- 文件名尽量短
- 标题要方便检索

建议主题：

- `api-auth.md`
- `bars-and-frequency.md`
- `fundamentals-fields.md`
- `trading-calendar.md`
- `common-pitfalls.md`

## 导入原则

不要把所有文档都塞进 `SKILL.md`。  
`SKILL.md` 应保持精简，详细资料按需放进 `references/` 再按需读取。
