# LangLang 股票工作区 AGENTS

这个工作区是 LangLang 的专用股票研究工作区，重点不是泛化，而是稳定、高密度、低废话。

## 启动顺序

进入股票任务前，按顺序读取：

1. `SOUL.md`
2. `USER.md`
3. `TOOLS.md`
4. `MEMORY.md`
5. `memory/YYYY-MM-DD.md` 中今天和昨天的记录

默认新会话先开启：

- `/verbose`
- `/reasoning`

## 语言规则

- 默认中文
- 结论和解释都用中文
- 命令、变量名、字段名保留原文

## LangLang 优先任务

- A 股研究
- 分钟级和日级历史分析
- 股票池筛选
- 板块联动判断
- 复盘和研究方案设计
- 用 JQData 反推触发结果背后的前置共性
- 根据研究结论生成 JoinQuant / JQData 回测代码

## 路由优先级

### 1. 历史研究与结构化筛选

当 `jqdata-research` 可用时优先使用：

- 股票池过滤
- 历史 K 线研究
- 分钟级研究
- 回测与验证
- 共性倒推研究

### 2. 聚宽工作流与文档约束

当任务涉及聚宽流程、研究设计、JQData 规则时，优先按 `jqdata-playbook` 的约束组织输出。

默认顺序是：

1. 先把用户给的结果条件转成研究问题
2. 再用 JQData 反推前置共性
3. 先输出共性和研究结论
4. 再按聚宽格式生成回测代码
5. 代码交给用户自己粘贴到聚宽回测

### 3. 外部能力补充

当 `qveris-official` 可用时优先使用：

- 发现新的数据或 API
- 本地未覆盖的数据能力
- OCR 或特殊外部能力
- 盘中选股
- 盘中实时筛选
- 通过 `qveris.ai` 调用同花顺工具能力的任务

注意：

- `qveris-official` 默认不是 LangLang 前置研究阶段的第一优先级
- 它主要用于“策略已认可后的盘中选股阶段”

### 4. 通用搜索

只有新闻背景、评论材料、说明性文档这类任务再用通用搜索。

## LangLang 输出要求

输出优先保持这五段：

1. 结论
2. 核心依据
3. 数据来源
4. 风险 / 失效条件
5. 下一步动作

如需更具体，补下面字段：

- 标的范围
- 时间范围
- 信号定义
- 研究限制

如果用户要求生成代码，输出顺序固定为：

1. 研究问题
2. 反推出的共性
3. 信号定义
4. 回测代码
5. 风险与限制

## 强制边界

- 没有 `jqdata-research` 时，不假装做过历史执行
- 没有实时数据时，不假装给盘中事实
- 没有验证时，不把想法写成事实
